# Инвентарь декор-приёмов — high-custom компоненты EntrancePage (Dragonia)

Что и как задекорировано в high-custom компонентах, и насколько это похоже/ново относительно нашего
«файла с трюками» (Vegas Hero: вектор-скос, opacity-стек свечения, тайл-текстура, металл-градиент).

## ГЛАВНЫЙ ВЫВОД — две школы декора в одной DS
- **Dragonia = школа РАСТРОВОЙ НАРЕЗКИ.** Рамки/кнопки/фоны собраны из заранее отрисованных кусков
  картинки, нарезанных на L/Middle/R (и несколько rect-пластин), `scaleMode=CROP`. Декор «запечён» в
  ассеты, а не построен процедурно.
- **Vegas Hero = школа ПРОЦЕДУРНОГО ВЕКТОРА/ЭФФЕКТОВ.** Вектор-полигоны (скос), стек полупрозрачных
  полигонов (свечение), тайл-текстура точек, многостоповые градиенты.
- Одна и та же DS (ForgeX), один контракт **3-слайс `part=Left/Middle/right`** (T1), но **противоположное
  исполнение**: baked-растр (Dragonia) против процедурного вектора (Vegas Hero).

## По компонентам

| Компонент | Приём | Знакомо/Ново |
|---|---|---|
| **Button** | 3-слайс `part=L/M/R` (T1), но пластины = **RECTANGLE + IMAGE `CROP`** (растровый, T2); лейбл/иконка = **градиент бело-золото `#EFCCAB` + тёмная обводка + жёсткая DROP_SHADOW** (T12) | Приёмы знакомы (T1/T2/T12); исполнение растровое (у Vegas Hero те же части были вектором) |
| **Frame-gamecard** (GameCard) | **Растровая 9-slice рамка** (T2): L/Middle/R, в каждом несколько RECTANGLE+IMAGE `CROP`; «Space» (opacity 0) = слот арта (негативное пространство) | T2 — знакомо; новое применение: рамка карточки как нарезанная картинка |
| **Frame-Sport** (SportCard) | То же растровое 9-slice (T2) + доп. слой «Border» IMAGE `FILL` поверх | T2 + image-overlay |
| **BG-casino-categories** | Фон меню = 3 куска IMAGE `CROP` (горизонтальная нарезка) | T2 для фона |
| **SpecialPromotionBanner** | Frame-Promotion-card = L/Middle/R (та же растровая нарезка) + Art-слот + Button(Secondary 3-слайс) | T1/T2 семейство |

| **Layout.Main.Background** | Растровый фон страницы | T2/raster |

## Совпадает с нашим trick-каталогом
- **T1** 3-слайс (контракт `part=Left/Middle/right`) — везде.
- **T8** градиент-sheen (MainBanner).
- **T12** градиентный лейбл с обводкой+тенью (Button — золотой текст).
- **T11** тень-отрыв (Button, Badge, ProgressBar — из прошлого diff).

## Новое / отличное от Vegas Hero
- **Растровая 9-slice как доминирующий приём** (T2) для рамок карточек, кнопок и фонов — «baked art»,
  а не процедурные вектора. Главное стилевое отличие школы Dragonia.
- **ArtGuide / Guide.Area** — оверлей безопасных зон под арт (danger/warning/default). Новая reusable
  находка, которой нет в каталоге; стоит занести как приём дизайн-процесса.
- **Золотой градиент-лейбл** (бело→`#EFCCAB`) с тёмной обводкой и hard-shadow — характерный brand-приём.

## Отсутствует в Dragonia (есть у Vegas Hero)
- Вектор-скос полигонами (T9-skew), opacity-стек свечения (L6), тайл-текстура точек (T7).

## Вывод для recon/сборки
- Распознавание Dragonia должно отдавать приоритет сигналу **IMAGE `scaleMode=CROP` в L/M/R-нарезке** →
  это «растровый 9-slice» (T2), а не процедурная сборка. `mapStrategy` для таких профилей должен
  рекомендовать **реюз нарезанных ассетов**, не перерисовку вектором.
- Добавить в каталог: **ArtGuide** (safe-zone оверлей) как отдельный приём.
