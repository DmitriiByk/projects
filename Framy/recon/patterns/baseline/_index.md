## Решённый блокер (для будущих проходов)
Figma MCP рвёт длинные/тяжёлые вызовы. Рабочий рецепт: **1 компонент за вызов**; для тяжёлых
(баннеры) — `figma.skipInvisibleInstanceChildren=true` + мелкая глубина обхода (d≤2). Загрузку
подграфа `getNodeByIdAsync` глубина не уменьшает — помогает именно skipInvisible.

## ВАЖНО для концепт-прохода (не пре-скипать утилитарные)
На чистом baseline инпуты/селекты/табы плоские → я их не профилировал. Но в КОНЦЕПТЕ они часто
декорируются (focus-glow, вдавленный фон + inner-shadow, иконные слоты, акцент-рамки). Поэтому при
diff против концепта профилировать ТАКЖЕ: Input (68639:2298), Select, Switcher, Tabs, Checkbox, Radio,
PinInput. Правило: на концепте дельту считаем по ВСЕМ компонентам, «плоское на baseline» ≠ «плоское в концепте».
