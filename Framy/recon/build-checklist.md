# Build-checklist — защита от повторяющихся промахов ребилда

Проходить ПЕРЕД тем, как объявить ребилд готовым. Каждый пункт рождён реальным промахом (L1–L8) и
закрыт либо кодом `faithful-rebuild.js`/`dumpSignals.js`, либо ручной проверкой. Корневое правило
сверху — остальное его следствия.

## Правило №0 — строй ИЗ ДАННЫХ, не по памяти
Геометрия, форма, opacity, цвета, пропорции, шрифт, текст лейбла — берутся из дампа узла, а не
выдумываются «правдоподобно». Любой параметр, заданный «на глаз», — это потенциальный промах.
Дефолт реконструкции — `recreate()` из `faithful-rebuild.js` (пересоздаёт из реальных значений).

## Чек-лист (пройти все)
1. **[L4] 3-слайс опознан** по part-наименованию / polygon-сегментам, центр FILL, L/R зеркальны.
2. **[L6] Свечение/объём = СТЕК полупрозрачных полигонов** (по реальным per-layer opacity), НЕ градиент.
   Форма слоёв — точная (пентагоны с плечами), не упрощённые треугольники. Зазоры — из данных.
3. **[нет чёрного контура]** у всех созданных vector/shape `strokes=[]`, если в оригинале их не было.
4. **[L5] База ≠ акцент.** Доминирующая заливка корпуса взята с МАСТЕРА декор-фона (не с одного рендера
   инстанса). Тёмный/светлый вид варианта — это override, не базовый цвет. Лейбл-цвет — по контрасту к
   реальной базе.
5. **[L3] Цель — декорированный мастер**, а не тонкая обёртка; при нужде провалиться в `mainComponent`.
6. **[L8] Шрифт проверен** через `listAvailableFontsAsync` ДО загрузки; нет — ближайший фолбэк (не падать молча).
7. **[L1/L2] FX-слой и текстура** распознаны (любой не-NORMAL бленд = FX; роль картинки база/оверлей).
8. **Пропорции и швы** из `relativeTransform`/размеров оригинала; слайсы стыкуются без шва и без лишней тени.
9. **[L9] Нет дефолтных белых заливок** у пересозданных контейнеров (GROUP→FRAME): `fills` задан явно
   ([] если в оригинале не было). Скан на видимый белый SOLID у фреймов, которых не было в оригинале.
10. **Сверка с оригиналом скриншотом** ОБЯЗАТЕЛЬНА до «готово»: снять `screenshot` ребилда и оригинала,
   сравнить силуэт, цвет базы, свечение, лейбл, и **нет ли лишних/белых элементов**. Расхождение —
   вернуться к Правилу №0.

11. **[L11] Компонентность + растяжение.** Если в профиле есть `reuse_candidates` — фон собран из
   ИНСТАНСОВ компонентов в auto-layout 3-слайсе (каплы FIXED, центр FILL), а не плоской геометрией.
   Проверить растяжение на ≥2 ширинах: центр растёт, каплы/острия на месте, симметрия держится.

12. **[L12] Структура снята ДО сборки** — auto-layout, слоты, арт, констрейнты реального инстанса (не «на глазок»).
13. **[L13] Реюз продакшн-компонента** как базы (инстанс/клон + наполнение слотов), если он есть.
14. **[L14] Реальный арт вставлен** (existing imageHash), обрезан по форме — ничего не торчит за рамку.
15. **[L15] Auto-layout + слоты**, не абсолют; тянется; нет overflow; проверено на ≥2 размерах.
16. **[L16] Подгружены профильные скиллы** (figma-generate-library / impeccable / image-asset-handler) перед сборкой.

## Быстрый путь (рекомендуемый)
1. `dumpSignals()` → профиль + флаги + `reuse_candidates`.
2. `mapStrategy()` → приёмы + режим (Слой 0).
3. **Если `reuse_candidates` есть → собрать из инстансов** (3-слайс, центр FILL, тянется) — доктрина
   «реюз > имитация». Иначе → `rebuildFaithful()` (плоская точная реконструкция, только для сверки).
4. Чек-лист 1–11 → скриншот-сверка + числовая проверка симметрии + тест растяжения.
