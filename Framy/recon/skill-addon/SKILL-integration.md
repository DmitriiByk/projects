# Интеграция recon-конвейера в скилл `build-game-ui`

> Перенос делается вручную через **Settings → Capabilities** (из cowork-сессии писать в установленный
> скилл нельзя — его файлы read-only). Ниже — что скопировать и какой текст вставить.

## Шаг 1 — скопировать файлы
Скопировать папку `references/recon/` (рядом с этим файлом) в скилл:

```
build-game-ui/references/recon/
  dumpSignals.js
  recon-map.js
  faithful-rebuild.js
  build-checklist.md
  lessons.md
  golden-primary-md-default.json
```

## Шаг 2 — вставить в `build-game-ui/SKILL.md`

### 2a. Новый раздел — сразу ПЕРЕД «## Порядок действий»
```markdown
## Recon-конвейер (распознать → стратегия → собрать) — делать ПЕРВЫМ
Не угадывать форму «на глаз». Перед «Порядок действий» прогнать:
1. `references/recon/dumpSignals.js` — снять стабильный JSON-профиль узла (флаги + `reuse_candidates`),
   с фолбэком резолва узла. Идемпотентно.
2. `references/recon/recon-map.js` → `mapStrategy(profile,{mode})` — профиль → приёмы T1–T15 + режим
   copy/learn (Слой 0) + tier-гейт (Rule 0).
3. **Сборка по результату:** если `reuse_candidates` есть → собрать из ИНСТАНСОВ компонентов в
   auto-layout 3-слайсе (каплы FIXED, центр FILL — ТЯНЕТСЯ); фон = инстанс компонента, не плоская
   геометрия. Иначе → `references/recon/faithful-rebuild.js` (`rebuildFaithful` — плоская точная
   реконструкция из данных, только для сверки/когда компонентов нет).
4. **Сдача** — `references/recon/build-checklist.md`: нет дефолтного чёрного контура; база≠акцент;
   свечение/объём = СТЕК полупрозрачных полигонов (per-layer opacity), не градиент; цвет базы — с
   мастера, не с одного рендера; шрифт проверен с фолбэком; **симметрия проверена числом**, не только
   скрином; **тест растяжения** на ≥2 ширинах.

Уроки-провалы + детекторы (копятся): `references/recon/lessons.md` (L1–L11). Каждый провал → детектор
в чек-листе. Золотой профиль-эталон: `references/recon/golden-primary-md-default.json`.
```

### 2b. В раздел «## Reference-файлы» добавить строки
```markdown
- `references/recon/dumpSignals.js` — recon-дамп узла → стабильный JSON-профиль (флаги, reuse_candidates).
- `references/recon/recon-map.js` — профиль → приёмы T1–T15 (`mapStrategy`), режим + tier-гейт.
- `references/recon/faithful-rebuild.js` — плоская реконструкция из данных (`rebuildFaithful`) для сверки.
- `references/recon/build-checklist.md` — preflight сдачи (Правило №0: строй из данных).
- `references/recon/lessons.md` — уроки-провалы L1–L11 с детекторами и регресс-кейсами.
```

### 2c. (опц.) В «Чек-лист сдачи» дописать пункты
Перенести «Кандидаты в чек-лист сдачи» из конца `lessons.md` (пункты L1–L11): нет чёрного контура;
роль картинки (база/оверлей); FX-бленд как слой; дамп мастера, не обёртки; база≠акцент; свечение
стеком полигонов; шрифт с фолбэком; нет дефолтных белых заливок; **компонентность + тест растяжения**.

## Что меняется в скилле по сути
- Разведка формы из «на глаз» → исполняемый дамп + детерминированное дерево приёмов.
- Сборка по умолчанию — **реюз компонентов (3-слайс, тянется)**, имитация только при отсутствии частей.
- Накопительная память провалов (L1–L11) с детекторами в чек-листе — промахи не повторяются.
