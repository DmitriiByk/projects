# OnePass → ForgeX: миграция токенов через Scripter + Tokens Studio

Цель: переименовать переменные OnePass под схему ForgeX **in-place** (сохраняя ID, чтобы привязки на компонентах не сломались), а затем синхронизировать токены через Tokens Studio **без создания дублей**.

Порядок строгий: сначала скрипт переименовывает переменные в Figma, и только потом импортируется JSON. Если сделать наоборот — Tokens Studio создаст параллельный набор, а компоненты останутся на старых переменных.

---

## Шаг 0. Бэкап (обязательно)

Переименование и перевод значений на алиасы — операции с продакшн-файлом. Перед началом:

- В Figma: правый клик по файлу → **Save to version history** (назвать, например, `pre-forgex-migration`), **или**
- Создать ветку: меню файла → **Create branch** (`forgex-token-migration`). Мигрировать в ветке, смотреть результат, потом мёржить.

Откат при проблеме: вернуться к версии из истории либо удалить ветку без мёржа.

---

## Шаг 1. Dry-run скрипта переименования

1. Открой файл **OnePass** (`Redesign-Application`) в десктопном Figma.
2. Установи плагин **Scripter** (автор Rasmus Andersson) — он даёт async-доступ к `figma.variables.*`.
3. Запусти Scripter, вставь весь код из `figma-migrate-variables.js`.
4. Убедись, что вверху файла `const DRY_RUN = true;` (значение по умолчанию).
5. Нажми **Run (▶)**. Скрипт **ничего не записывает**, только печатает план: что переименует, какие алиасы поставит, что создаст, и список `WARN` (не найденные имена).
6. Прочитай отчёт внизу лога. Норма: часть `WARN` по партнёрским именам (`Brand Colors/…`) и по токенам, которых в этом файле нет.

Что проверить в логе dry-run:
- число «Переименовано» близко к ожидаемому (примитивы + семантика);
- нет переименований, которые ведут к коллизии имён (две переменные в одно имя);
- целевые переменные для алиасов находятся (мало `WARN` в секции алиасов).

---

## Шаг 2. Применение скрипта

1. Только после удачного dry-run поставь `const DRY_RUN = false;`.
2. Нажми **Run (▶)** ещё раз.
3. Скрипт выполнит пять шагов: (1) переименует переменные in-place (ID сохраняются → привязки живут), (2) создаст недостающие `theme/*` (`base`, `inverse`, недостающие `main/*`), (3) проставит семантические алиасы, (4) переведёт легаси-короткие имена на алиасы к `palette/*`, (5) **переименует текст-стили типографики** (`display/xl → typography/display/xl`, `bodyRegular/sm → typography/body/sm` и т.д. — это отдельный Figma API, значения стилей не трогаются).

> Примитивы типографики (`fontFamily/* fontWeight/* fontSize/* lineHeight/*`) уже соответствуют неймингу ForgeX — скрипт их НЕ трогает. Spacing / radius / shadows в OnePass не токенизированы (переменных нет) — в этот проход не входят, это отдельный проект.

> Перевод значения переменной на алиас вызывает ре-рендер привязанных слоёв — это нормально и ожидаемо. Скрипт идемпотентен: повторный запуск пропустит уже переименованное.

После применения имена переменных в Figma совпадают с путями в `onepass.tokens-studio.json`. Теперь Tokens Studio будет матчить их по имени и работать in-place.

---

## Шаг 3. Импорт Tokens Studio (синхронизация, не дубли)

1. Установи плагин **Tokens Studio for Figma** (бесплатный), открой в файле OnePass.
2. Импорт: **Tools → Load from file/folder** (или вкладка Settings → Import) → выбери `onepass.tokens-studio.json`.
   - Файл содержит наборы: `palette`, `semantic-light`, `brand-partners`, `typography`, `legacy` (порядок задан в `$metadata.tokenSetOrder`) и тему `OnePass · Light` в `$themes`.
3. Активируй тему **OnePass · Light** (вкладка Themes), чтобы включились нужные наборы.
4. Экспорт в переменные: **Settings → включи «Create/Update Figma variables»** и в опциях экспорта выбери:
   - **Sync existing variables by name** (обновлять существующие по совпадению пути) — НЕ создавать новые, если имя уже есть;
   - сопоставление наборов с коллекциями: `palette` → коллекция примитивов, `semantic-light` → коллекция темы, `brand-partners` → отдельная коллекция партнёров;
   - режим (mode) — тот же активный, что использовал скрипт (default mode файла);
   - **типографику в Tokens Studio НЕ включай** (отключи набор `typography` при экспорте). Текст-стили уже переименованы скриптом in-place; если дать их ещё и Tokens Studio — получишь дубли стилей.
5. Запусти экспорт. Поскольку имена уже переименованы скриптом, Tokens Studio обновит существующие переменные, а не наплодит дубли.

---

## Шаг 4. Пост-чек

Открой и глазами проверь:

- **Discovery** и **Casino page** — цвета не поехали, тёмно-синий бренд (`#0080FF`) на месте, текст читаем.
- В панели переменных: **нет двойных коллекций** (`Pallette` и `palette` одновременно), нет осиротевших дублей.
- Выбери пару компонентов (`item-casino`, кнопки) → у привязок к цветам новые имена (`palette/*`, `main/*`, `text/*`), привязка не отвалилась в «detached».
- Легаси-короткие имена остались, но указывают алиасом на `palette/*` (помечены deprecated).
- Партнёрские бренды — в отдельной коллекции `brand-partners`, не в ядре.
- `additional/550 = #00D4FF` (циан) на месте в группе `additional`.
- Inter в Paragraph не тронут.

Если что-то поехало — откат по Шагу 0 (вернуть версию / удалить ветку).

---

## Что входит в этот проход

- Цвета: `palette → tint/main/text/base/inverse/links` + легаси-алиасы + изоляция партнёрских брендов.
- Типографика: переименование текст-стилей под `typography/<group>/<size>` (`bodyRegular→body`, `bodyMedium` сохранён как расширение). Примитивы типографики уже соответствуют ForgeX.

## Границы (follow-up, НЕ входит)

- **Spacing / radius / shadows** — в OnePass не токенизированы (переменных нет вообще, значения захардкожены). Это отдельный проект: вывести шкалу из хардкода → завести переменные → перебиндить компоненты. Переименовывать тут нечего.
- Группа `label` ForgeX — в OnePass отсутствует, не создавалась.
- Решения владельца уже учтены: `neutral/400 = #8C87B3`; `Background/tertiary` — ближайший примитив (дрейф `#EEECFF→#D3C9EF`); расширенные ключи `text/*` и `bodyMedium` сохранены (OnePass — отдельный продукт); Inter не тронут.
