# Ревью: figma-migrate-variables.js + tokens-studio-import-guide.md

**Вердикт: PASS с замечаниями** (1 блокер, 3 важных, 4 мелочи)

Скрипт архитектурно грамотный: DRY_RUN-гарды стоят перед всеми тремя операциями записи, удалений нет, маппинг семантики на 100% совпадает с JSON (38/38 алиасов), идемпотентность продумана. Но есть один реальный блокер-коллизия из спека, отсутствующий гард modeId на ШАГе 4 и вводящее в заблуждение поведение dry-run.

| Severity | Строки | Проблема | Правка |

| **БЛОКЕР** | 159, 162 | **Коллизия: два токена → одно имя.** `Links/Alert` и `Functional/Error` оба → `theme/main/negative/base`. По спеку (стр. 151/154/281) это ДВА отдельных токена («оба токена сохранены»). Скрипт переименует первый, второй упадёт в WARN «target busy» (стр. 429) и останется со старым именем `Functional/Error` — НЕ мигрирован. Потеря семантического токена / расхождение со спеком. | Дать `Functional/Error` отдельное имя (напр. `theme/main/negative/error`) и добавить его в ALIAS_MAP → `palette/negative/500`. | **Важное** | 512-516 | **Нет null-гарда modeId на ШАГе 4** (LEGACY). В ШАГе 3 есть `if (!modeId)` (484-488), а тут нет. Если коллекция легаси-переменной не в `collById`, `setValueForMode(null, alias)` упадёт в APPLY. | Добавить перед записью: `if (!modeId) { console.log(...); report.warns.push("legacy-no-mode:"+legacyName); continue; }` |

Dry-run не отражает реальный план для алиасов. | Для dry-run строить виртуальную карту переименованных имён (old→new) и резолвить semName/targetName через неё; либо явно отметить в логе, что секции алиасов в dry-run неинформативны. |
| **Важное** | 369-377 | **createVariableCompat: try/catch может маскировать реальную ошибку.** Если новый API создаст переменную, но затем бросит по другой причине, catch сделает ВТОРОЙ `createVariable` → дубль. ## Что проверено и ОК

- Удалений нет; всё вне карт не трогается.
- Все 3 операции записи закрыты `if (!DRY_RUN)`.
- modeId берётся из коллекции САМОЙ переменной (`defaultModeId`), не чужой.
- 38/38 семантических алиасов точно совпадают с `onepass.tokens-studio.json` (пути и palette-цели).
- Все theme/*-таргеты RENAME имеют запись в ALIAS_MAP; нет осиротевших ключей; все palette-цели алиасов существуют в RENAME.
- Партнёрские «дубли» (две формы имени) — ожидаемые, не коллизия.
- Идемпотентность: `already`/`exists`/`already.id===src.id` обработаны.
- APPLY-refresh после rename (447) и push созданной переменной (461) корректны.
- Гайд: порядок «скрипт → TS», DRY_RUN, Scripter, «sync by name / не плодить дубли» — описаны верно.
