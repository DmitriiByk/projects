# Framy — панель управления сабагентами Claude Code

Веб-приложение, которое показывает всех ваших сабагентов Claude Code, позволяет
сменить модель агента, дать ему задачу и запустить её, а также добавить новых
агентов — вручную, импортом из GitHub-репозитория или вставкой markdown.

```
framy-agents/
├─ server/   # Express API: читает/пишет .claude/agents, запускает claude CLI, импорт с GitHub
└─ web/      # React + Vite интерфейс
```

## Что умеет

- **Видеть всех агентов** — сканирует `~/.claude/agents` (глобальные) и `.claude/agents`
  в каталоге проекта. Карточка показывает имя, описание, инструменты, область и модель.
- **Сменить модель** — выпадающий список (Inherit / Opus / Sonnet / Haiku) прямо на карточке;
  значение записывается в `frontmatter` файла агента.
- **Дать задачу** — кнопка «▶ Дать задачу» открывает панель: вводите задачу, при желании
  переопределяете модель, жмёте «Запустить». Бэкенд вызывает `claude -p …` с делегированием
  выбранному сабагенту и стримит вывод в реальном времени (SSE).
- **Добавить агента** тремя способами:
  - **Вручную** — форма (имя, описание, модель, инструменты, системный промпт).
  - **Из GitHub** — вставляете ссылку на репозиторий (или `owner/repo`); приложение находит
    `.md`-агентов в `.claude/agents`, `agents/` или корне, показывает предпросмотр,
    вы выбираете нужных и импортируете. Есть быстрые ссылки на популярные коллекции.
  - **Вставить markdown** — вставляете содержимое `.md`-файла агента целиком.
- **Удалить агента** — удаляет файл с диска (с подтверждением).

## Требования

- Node.js 18+ (нужен встроенный `fetch`).
- Для **запуска задач** — установленный [Claude Code](https://docs.claude.com/en/docs/claude-code)
  с доступной в `PATH` командой `claude`. Без него остальные функции работают, а запуск
  покажет понятную ошибку.

## Установка и запуск

```bash
cd framy-agents
npm install          # поставит зависимости и для server, и для web
npm run dev          # поднимет API (порт 5174) и UI (порт 5173) одновременно
```

Откройте <http://localhost:5173>.

### Продакшн-сборка

```bash
npm run build        # собирает web в web/dist
npm start            # сервер на 5174 раздаёт и API, и собранный фронтенд
```

## Настройка каталогов и окружения

Переменные окружения для `server`:

| Переменная | Назначение |
|---|---|
| `FRAMY_AGENT_DIRS` | Каталоги агентов через `:` (первый — куда писать новых). Переопределяет дефолт. |
| `FRAMY_PROJECT_DIR` | Корень проекта, чей `.claude/agents` подхватывать (по умолчанию — cwd сервера). |
| `FRAMY_CLAUDE_BIN` | Путь к бинарю `claude`, если он не в `PATH`. |
| `GITHUB_TOKEN` | Токен GitHub — поднимает лимит API при импорте из приватных/частых репозиториев. |
| `FRAMY_INSECURE_TLS` | `1` — не проверять TLS-сертификаты при обращении к коннекторам (для корп-прокси с перехватом TLS). Только в доверенной сети. |
| `FRAMY_CLAUDE_CREDS` | Путь к файлу креденшелов Claude Code, если токены не в Keychain. |
| `PORT` | Порт сервера (по умолчанию 5174). |

Пример — смотреть агентов конкретного проекта:

```bash
FRAMY_PROJECT_DIR=/path/to/my-project npm run dev
```

## Формат файла агента

Claude Code хранит каждого сабагента как markdown с YAML-frontmatter:

```markdown
---
name: security-reviewer
description: Use this agent to review code for security vulnerabilities
model: sonnet
tools: Read, Grep, Bash
---

Ты — старший инженер по безопасности. Анализируй изменения и указывай риски…
```

Framy читает и пишет именно этот формат, поэтому изменения сразу видны и самому
Claude Code, и наоборот.

## Как работает запуск

Сабагенты вызываются не напрямую, а через главного агента. Framy формирует промпт
вида «Use the "<name>" subagent to handle the following task…» и выполняет
`claude -p "<промпт>" [--model <model>]` в нужной рабочей директории, передавая вывод в UI.
