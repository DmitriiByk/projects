# Каталог приёмов T1–T15 (точные значения)

Формат: суть → механизм/значения → где. Палитры — примеры из разобранных кнопок;
для своей кнопки **бери реальные значения из файла** (см. железное правило в SKILL.md).

## T1. Масштабируемый 3-слайс
Горизонтальный auto-layout; края `layoutSizingHorizontal="FIXED"`, центр `"FILL"`, высота `"FILL"`.
Фон-3-слайс — абсолютный (`layoutPositioning="ABSOLUTE"`, `constraints {STRETCH,STRETCH}`), под лейблом.

## T2. 9-slice растра со швами по пикселям
Каждый слайс — прямоугольник:
`fills=[{type:"IMAGE",scaleMode:"CROP",imageHash,imageTransform:[[cw,0,L],[0,1,0]]}]`.
Соседние слайсы делят ОДИНАКОВУЮ долю-границу. Пример cap=0.178: left `L=0,cw=.178`;
center `L=.178,cw=.644`; right `L=.822,cw=.178`. Центр FILL, капы FIXED → центр тянется, швы не уезжают.
Один `imageHash` на все три слайса = минимум памяти.
Анти-паттерн: `FILL`, либо `CROP` с `imageTransform`, натягивающим фикс-долю на всю ширину → растягивает.

## T3. Глубина alpha-градиентами + wash (неон/стекло)
Слои центра (снизу вверх): база `#a9f29d` → wash `#a9f29d @0.30` → тёмная фаска градиент `#121723`
альфа `[1@0, 0@0.47, 1@1]` (+ INNER_SHADOW) → точки (тайл-PNG) → неон градиент `#6fea5c`
альфа `[1@0, 0@0.19, 0@0.76, 1@1]`. Токены Vegas Hero: `#6fea5c / #a9f29d / #c5f7be`.

## T4. Гладкий глянец
Вертикальный градиент светлый верх→тёмный низ + верхний блик + нижняя фаска + INNER_SHADOW.
Пример (зелёная): `#e9ffd0→#b9f78d→#74e646→#47b82f→#2c8a22`; блик белый `0.75→0` сверху;
фаска тёмная `0→0.5` снизу.

## T5. Объём «пилюли» (CasinoLab)
Pill (`cornerRadius`=½H), мягкий градиент, тонкий белый `stroke #ffffff@0.20 w1 INSIDE` сверху,
**жёсткая тень снизу** `DROP_SHADOW off(0,5) blur0` цветом темнее заливки (+ опц. мягкая `blur15 a0.4`).
Темы: orange `#ffe11f→#ff8533` hard `#e36d1f`; blue `#cddcfb→#9bbaf8` hard `#588cf3`;
purple `…→#cbbcff` hard `#a88fff`; ghost `#ffffff→#eef3fe` hard `#dee8fd`.

## T6. Радиальное ядро свечения (Cazeus)
Радиал яркий центр→тёмные края + INNER_SHADOW + тонкий металл-`stroke`. Значения:
radial `#1b6dff→#1757ce→#14419d→#102a6c→#0e1f54→#0c143b`; border `#ffb800` (бронза `#cd7f32`);
inner `inset 0 4 4 #919bf3` + `inset 0 -3 6 rgba(102,157,255,.6)`.
Радиал точнее задавать через `createNodeFromSvg` (`<radialGradient cx=50% cy=44% r=62%>`).

## T7. Энергетическая текстура (растр + blend + маска)
Растр с `mix-blend-mode:LIGHTEN`, обрезать alpha-маской по форме, резать Left/Middle/Right для масштаба.
Тёплый металл × холодная энергия: бронза + синяя молния — гармонично.

## T8. Многослойная металлическая рамка
Стопка концентрических rounded-rect, у каждого свой градиент (constraints STRETCH, инсеты постоянны):
внешний тёмный кант `#7a5618→#2e1d06` → основной металл `#ffe98a→#f0c040→#a9760f` (inset ~1.5) →
внутренний светлый кант `#fff6cf→#caa033` (inset ~3.5) → цветная панель (inset ~5.5).
Рейлы (если рамка из капов) — `strokeTopWeight/strokeBottomWeight`, бока 0, обводка-градиент `#ffb800→#5b2c15`.

## T9. Орнамент-капы + зеркало
Орнамент — отдельный вектор-компонент; правый = левый зеркалом `relativeTransform=[[-1,0,x+w],[0,1,0]]`.
Рисовать по `vector-method.md`. Гладкое золото — 8-стоповый градиент с резким переломом ~0.5.

## T10. Блики на фоне (glints)
(1) верхний глянец — белый `0.55→0` у верха; (2) диагональная засветка — повёрнутый ~20° узкий
прямоугольник белый `0→0.28→0`; (3) INNER_SHADOW белый сверху + тёмный снизу.

## T11. «Выдавить из фона» + нижний блик
- Внешняя тёмная тень (две): мягкая `#000 a0.7 off(0,6) b16` + плотная `#000 a0.55 off(0,0) b10 s1`.
- Нижний блик ТОНКО: `ELLIPSE` (широкий, h≈14, ~58% ширины) у низа, тёплый SOLID `opacity~0.4` +
  `LAYER_BLUR ~6`, `constraints {CENTER, MAX}`. Жёсткий градиент-бар во всю ширину = грубо.

## T12. Градиентная заливка ТЕКСТА
TEXT-нода: `fills=[GRADIENT_LINEAR]` (light верх→акцент низ) + `effects=[DROP_SHADOW off(0,1) blur0 #000]`
(эмбосс). Пример: `#ffffff(0–40%)→#b1ff9a`; шрифт Fira Sans Condensed ExtraBold, uppercase, tracking ~2%.

## T13. Анимация только на hover (GIF-фон)
Фон-«лицо» = анимированный GIF (без текста), вшит через `figma.createImage`. Текст — отдельный
Figma-слой. Компонент-сет State=Default(статичный кадр)/Hover(GIF). Реакция на Default:
`trigger ON_HOVER → action CHANGE_TO Hover` (while-hovering авто-возврат). Узел сам обрезает GIF
по rounded-форме (clipsContent). Код — `figma-recipes.md`.

## T14. Состояния
Один скелет, меняем свет/насыщенность/glow/наличие FX. Hover — ярче + внешний glow; Active — темнее
+ «вдавленный» INNER_SHADOW + FX ~50%; Disable — обесцвечено, FX off, `opacity~0.65`.
Держать вариантами компонент-сета. Иерархия Type: Secondary/Tertiary = де-акцент (тише палитра/блик),
но тени и блики остаются (мокко-secondary: рама `#9a7450→#4a2e1c`, текст white→`#c3a9a5`).

## T15. Текстура: форма эмерджентная, смотреть ОБЩЕЕ
Узор из «точек» может складываться в стрелки; бывает несколько КРУПНЫХ направленных форм
(слева→, справа←, симметрично), а не равномерное поле. Алгоритм: (1) увидеть макро-композицию
(сколько форм, где, направление); (2) силуэт формы; (3) примитив (точка) и плотность.
Респонсив: текстура НЕ тянется — фикс. сетка/группы с `constraints:MIN`/`MAX` к краям; база тянется.
Воспроизводить все три уровня, а не один (типичный промах: «горошек» вместо нескольких стрелок).

## T16. Вогнутые угловые вырезы (boolean SUBTRACT) + многослойная глубина
Каменные/металлические слэбы с «вычерпнутыми» углами (бортик-плита). НЕ скругление — углы ОСТРЫЕ (r=0).
- Вырез = прямоугольник МИНУС объединение 4 эллипсов, центры которых стоят ТОЧНО на углах
  прямоугольника (`ellipse.x=cornerX - D/2`, `ellipse.y=cornerY - D/2`). Круг, центрированный на углу,
  убирает четвертину → вогнутый scoop; между вырезами угол остаётся острым 90°. Ставить по формуле, не на глаз.
- Boolean: `figma.union(ellipses, parent)` → `figma.subtract([rect, union], parent)`. Материал-заливку
  (stone IMAGE + тёмный SOLID-тинт для канта) и эффекты вешать на УЗЕЛ boolean-результата, не на детей.
  Эллипсы-резаки — любой fill (он не виден). `INNER_SHADOW` на результате = тёмная каёмка среза (карвинг).
- Многослойная глубина (КЛЮЧ): у КАЖДОГО Z-слоя СВОЙ набор резаков. Нижний/кант — эллипсы меньше (D≈16),
  верхняя приподнятая плита (инсет ~4) — КРУПНЕЕ (D≈24). Срез уходит «ступенькой» вглубь: в углу видно
  тёмную дыру → каменный кант → лицо. Лицо приподнято `DROP_SHADOW` вниз (выше по Z), кант притемнён
  и «утоплен» `INNER_SHADOW`. На весь компонент — внешняя тень-отрыв.
- РОВНЫЙ кант (одинаковый и на прямых, и в углах): вырезы соседних слоёв ДЕЛАЮТСЯ КОНЦЕНТРИЧНО —
  один центр (в углу ПАНЕЛИ, не в углу инсетнутого слоя!), радиусы отличаются ровно на ширину канта B.
  Кант: r=Ro; лицо (инсет B): r=Ro+B, центр ТОТ ЖЕ. Тогда зазор между дугами = B везде = ширине прямого
  канта. Частый промах: резак лица центрить в углу лица (инсет) — тогда в углу кант шире, чем по бокам.
- РЕСПОНСИВ (лучший способ — ОДНИМ boolean-слоем; текстура непрерывна, без швов):
  • родитель НЕ auto-layout (в auto-layout у boolean нет `constraints`).
  • сам boolean НЕ трогать: у `BOOLEAN_OPERATION` свойства `constraints` НЕТ — присвоение бросает ошибку.
  • респонсив задают ДЕТИ внутри boolean: базовый rect — `SCALE` (тянется вместе с углами) либо `STRETCH`
    (фикс-инсет для лица); 4 кружка-резака ПИНЯТСЯ каждый к своему углу (`MIN/MIN, MAX/MIN, MIN/MAX, MAX/MAX`)
    фикс-размера. При ресайзе boolean пересчитывается: кружки остаются КРУГЛЫМИ в углах при любой пропорции.
  • камень — один `scaleMode:TILE` fill на boolean → непрерывный, без швов при любом размере.
  • кант/лицо = два таких boolean-слоя: кант (full-size, stone+тёмный тинт, rect SCALE, R=Ro) и лицо
    (инсет B, чистый stone, rect STRETCH, R=Ro+B концентрично). Этим же приёмом лепятся ЛЮБЫЕ отзывчивые шейпы.
  Запасные пути (если нужно): 3-слайс (тянем по ширине) или 9-слайс solid-МАСКА над непрерывным камнем
  (обе оси) — тяжелее. НЕЛЬЗЯ нарезать САМ камень на секции с TILE — будут швы тайла. Тяжёлую внешнюю
  тень-отрыв легко переборщить — лёгкий контактный след или убрать.
- Применимо к панелям/хедерам/карточкам/бейджам.

## T17. «Кожаная» кнопка: расширяемое тело + строчка-рамка + угловые заклёпки + гравировка
Разбор кожаной кнопки (тиснёная кожа, белая строчка, металлические уголки, золотой текст).
- ТЕЛО — горизонтальный 3-слайс ОДНОГО растра кожи (контейнер `overflow-clip` + rounded): левый кап FIXED
  ~13px / центр FILL / правый кап FIXED ~12px; все три — один `imageHash` с разным `imageTransform` (CROP).
  Тянется по ширине, зерно не растягивается и непрерывно (см. T2).
- МЯГКИЙ ВНУТРЕННИЙ КАНТ (`sh`) — отдельный rect: тёмная обводка (`border 2px #160c03`) + rounded +
  `LAYER_BLUR ~0.9`, инсет ~7px. Размытая ОБВОДКА = мягкая внутренняя фаска/тень по скруглению
  (живая альтернатива INNER_SHADOW, точно следует радиусу).
- СТРОЧКА (Thread) — декоративная РАМКА как auto-layout-кольцо из ТАЙЛЕННЫХ растров-ниток + угловые куски:
  Top (уголок + горизонт. нить `TILE` FILL + уголок) / Center (две вертик. нити `TILE`, `justify-between`/пин
  к краям) / Bottom (уголок + нить + уголок). Нить — мелкий растр (`bg-size` = один стежок), ТАЙЛИТСЯ вдоль
  кромки (масштаб = повтор, не растяжение) → отзывчивая обводка-строчка.
- УГЛОВЫЕ ЗАКЛЁПКИ — ОДИН ассет переиспользован 4× зеркалом/поворотом (`rotate 180`, `scaleX -1`), каждый
  запинен в свой угол; направленная DROP_SHADOW на КАЖДЫЙ: верхние — вниз (0,+1), нижние — вверх (0,−1)
  (#000@0.58, r1) → металл читается приподнятым при едином верхнем свете. (1 ассет → 4 угла.)
- ТЕКСТ — гравировка: тёплое золото SOLID `#e1c692` (шрифт Lora Bold, uppercase) + тёмная DROP_SHADOW
  `(0,1) r1 #231910` = тиснение по коже. Проще градиентного текста [T12] — один цвет + тёмный эмбосс.
- Слоты Start/End-иконок по бокам лейбла, паддинг тела ~6px.
- ИДЕЯ: «материальная» кнопка = расширяемая текстура (3-слайс CROP) + декор-рамка из тайлов + переиспользуемые
  угловые ассеты с направленными тенями + текст-эмбосс. Тот же каркас годится для дерева/металла/камня.
- РЕПЛИКА СВОИМИ МАТЕРИАЛАМИ (проверено): кожа — Pillow (worley-пузыри + эмбосс), `TILE` (не 3-слайс — проще,
  для гориз. ресайза достаточно). Упрощения, которые хорошо читаются: строчка — `dashPattern` на обводке
  (cream, инсет, rounded) вместо тайл-нитей; внутренний кант — обводка + `LAYER_BLUR` (sh); угол — ОДИН растр,
  4× через `scaleMode:CROP`+`imageTransform`-флип (X/Y), пины `MIN/MAX`, тени направлены (верх вниз/низ вверх);
  текст — Lora Bold, золото `#e1c692` + тёмная drop-тень. ГРАБЛИ: длинный base64 (>~6000) при ручной вставке
  бьётся → картинка чёрная; делай растр компактным (≤~4000 b64) или дроби на 2 вызова (см. figma-recipes).

## T18. Декомпозиция ЗАПЕЧЁННОГО растра в отзывчивый компонент (CROP + exportAsync)
Дано: одна плоская картинка-кнопка (кожа + плетёная ВЕРЁВКА по периметру + углы). Надо сделать отзывчивой —
разложить на слои и заставить верёвку ПОВТОРЯТЬСЯ, а не растягиваться.
- КАК «ВЫРЕЗАТЬ» КУСОК ИЗ КАРТИНКИ БЕЗ ВНЕШНИХ ТУЛОВ (ключ): `download_assets`/`web_fetch` дают URL на
  figma-домене, который песочница не качает (прокси). Вместо этого — РЕНДЕР куска внутри Figma:
  создать временный rect, залить ИСХОДНИК `scaleMode:"CROP"` c `imageTransform`, отдающим нужный регион
  `[sx,sy,sw,sh]` (в координатах отображения, нормировка на W×H исходника):
  `imageTransform=[[sw/W,0,sx/W],[0,sh/H,sy/H]]`; затем `bytes=await tmp.exportAsync({format:"PNG",
  constraint:{type:"SCALE",value:2}})` → `figma.createImage(bytes).hash`. Теперь это САМОСТОЯТЕЛЬНАЯ
  картинка региона, которую можно ТАЙЛИТЬ (CROP такого не умеет — он растягивает кроп). `tmp.remove()`.
- ЛОГИКА ВЕРЁВКИ: вырезать ОДИН период плетёнки (узкий юнит вдоль кромки), а не всю кромку. Юнит залить
  `scaleMode:"TILE"` в кромочную ячейку 9-слайса → при росте кнопки верёвка ПОВТОРЯЕТСЯ (не тянется).
  Верх/низ — гориз. юнит (TILE, ячейка FILL по ширине); левый/правый — верт. юнит (высота фикс → без растяжения);
  4 УГЛА — отдельные вырезанные куски, FIXED (петля/изгиб верёвки сохраняется точно).
- СЛОЙ КОЖИ отдельно: кроп кожи из исходника при TILE даёт ШВЫ (патч не бесшовный). Решение — центр залить
  СВОЕЙ бесшовной кожей (FFT/worley-тор, T-«камень»-метод) В ТОН исходника. Верёвка-герой остаётся из картинки.
- СБОРКА: 9-слайс auto-layout (углы FIXED / кромки FILL по одной оси + TILE / центр FILL+TILE) — как в T16-респонсиве.
  exportAsync-картинки рендерятся в `get_screenshot` (статика), проверяется на скрине. Период юнита подбирать
  так, чтобы шов тайла был незаметен (на плетёнке мелкий шов прощается).

## T19. Скошенная (параллелограм) неоновая CTA + простая глянцевая CTA
- ПАРАЛЛЕЛОГРАМ, который тянется без искажения скоса → 3-слайс: левый/правый ВЕКТОР-торцы (трапеции со
  скошенной внешней кромкой, FIXED) + сплошной центр FILL. Скос только в торцах, центр прямой → при росте
  ширины угол скоса постоянен. Торцы — `createNodeFromSvg` полигон (capL `M S 0 L W 0 L W H L 0 H Z`, capR зеркально).
  `itemSpacing:-1` чтобы убрать щель между слайсами.
- НЕОН-GLOW: цветная `DROP_SHADOW` (цвет заливки, blur~12, spread~1, ~0.5) на самих фигурах (не на прямоуг.
  фрейме — иначе ореол прямоугольный). Можно отдельным размытым слоем-дублёром позади.
- КРОМОЧНЫЙ БЛИК неона: тонкий (h~2) прямоугольник с ГОРИЗ. градиентом `transparent→свет→transparent` +
  `LAYER_BLUR 1`, прижат к верх/низ (`constraints` MIN/MAX), по ширине STRETCH, отступ от торцов.
- ТЕКСТ: тёмный на ярком фоне + мягкая ЦВЕТНАЯ тень-glow в тон фона; жирный курсив (Archivo Black Italic).
- ПРОСТАЯ ГЛЯНЦЕВАЯ CTA (красная): вертикальный градиент (ярче сверху→темнее снизу, `#ad0000→#4e0e0e`) +
  тонкая яркая обводка в тон (`#ff3030` ~0.75) + верхний ГЛЯНЕЦ (белый градиент `0.16→0` в верхней половине) +
  белый курсив (Archivo Bold Italic) с тёмной эмбосс-тенью `(1,1) blur4`. Всё `constraints:STRETCH` → тянется.

## T22 — Объём клавиши старого компьютера (outset-фаска)
НЕ плоский верт.градиент и НЕ боковой цилиндр. Объём = бевел клавиши: свет сверху-слева, тень снизу-справа.
Эффекты: INNER_SHADOW свет `#fffaf0`@0.95 offset(+2,+2) r1; INNER_SHADOW тёмн `#6b4a28`@0.7 offset(-2,-2) r2;
DROP_SHADOW отрыв `#251404`(0,4,r5,-1). Fills: база `#e3cdaa` + лёгкий vGrad верх светлее/низ темнее. Stroke `#5a4528`, r5.
Диагональные свет/тёмн INNER_SHADOW в разные углы = универсальный outset-bevel (меняй цвета канта под материал).
Иконку реюз (exportAsync icon-view-box @3x → FIT). Компонент+инстансы: оверрайд fill иконки + characters.
⚠️ layoutSizingVertical=FILL только ПОСЛЕ append в авто-лейаут.

## Дополнения-примеры (по правилу каталога — НЕ новые Txx)
- [к T1] 3-слайс БЕЗ auto-layout — через constraints: капы пиннятся (левый horizontal=MIN, правый=MAX, оба vertical=STRETCH), середина horizontal+vertical=STRETCH. Так вектор-капы ЛЮБОЙ формы тянутся по ширине без искажения торцов. Правый кап = левый зеркалом через SVG `<g transform="translate(w,0) scale(-1,1)">…</g>`.
- [к T8/материал] «Secondary/ghost» рамка-кнопка (матовый gunmetal): глубину несут ТОНКИЕ канты, не градиент — концентрика rim `#929292` → groove `#000` → rail `#929292` → панель INSET `#1e1e1e`+текстура. Де-акцент = низкая амплитуда (модель объёма п.5), но канты обязательны.
- [к T12] Гравировка текста (intaglio) на тёмном: стопка тёмных DROP_SHADOW с растущим offset/blur (`#000@.5 0/1/2`, `#000@.4 0/3/0`, `#000@.25 0/8/4`) + низкоконтрастная заливка (`#b4b4b4`).
- ⚠️ РАЗБОР ОШИБКИ (Secondary 'LOG IN' 169039:15582): принял форму за прямой 8-угольник по тхамбнейлу 92×36 и нарисовал прямыми фасками → ОТРИЦАТЕЛЬНО. На деле рамка ОРНАМЕНТАЛЬНАЯ: ogee-кривые + угловые завитки (те самые мелкие `Vector` 3–6px в дампе) + текстурная панель (`IMAGE`-заливка). Урок: непрямоугольную форму НЕ аппроксимируй — вытащи реальный `vectorPaths` (клон слоя-контура) и реюзай текстуру панели (exportAsync). Проверяй форму на УВЕЛИЧЕННОМ скрине, а сигнал орнамента — в дампе (см. Железное правило).
