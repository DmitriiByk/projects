# build-game-button

Сборка/репликация стилизованных кнопок в Figma. Главная ценность скилла — **дерево решений**:
кнопок много и собираются они по-разному, поэтому сначала классифицируем кнопку, потом
выбираем приём(ы) из каталога и строим. Точные рецепты и значения — в `references/`.

## Железное правило (нарушать нельзя)
**Бери факты из файла, не угадывай.** Прежде чем строить — сделай ДАМП реальных `fills`/
`opacity`/`blendMode`/`effects`/`strokes`/`cornerRadius`/`vectorPaths` оригинала через
`use_figma` и сними скриншот **в зуме на КОНТРАСТНОМ фоне**. Внешние тени, нижние блики и
форму текстуры легко не заметить в мелком тёмном тхамбнейле. ⚠️ ФОРМА — не угадывать по тхамбнейлу: нативный скрин мелкой ноды (напр. 92×36) СКРЫВАЕТ орнамент (завитки читаются как «насечки»). Снимай скрин В УВЕЛИЧЕНИИ (нода/инстанс ≥3×) И читай сигналы ДАМПА: мелкие угловые Vector 3–6px = орнамент-завитки; IMAGE в заливке = текстура (реюзай exportAsync); в vectorPaths команды C = кривые/ogee, не прямые фаски. Непрямоугольную/орнаментальную форму НЕ аппроксимируй прямыми/rounded — БЕРИ реальный vectorPaths (клон слоя-контура или восстанови путь, T17/vector-method). Нельзя считать вариант
«упрощённым» и срезать объёмные приёмы — де-акцент это только палитра/насыщенность.

## Порядок действий

1. **Доступ/формат.** Есть Figma node-ссылка (читаю слои/значения) или только картинка
   (визуальный разбор)? Финал — Figma.
2. **Разведка.** `get_metadata` → `get_design_context` → `get_variable_defs` → дамп слоёв
   (см. `references/figma-recipes.md` → «Дамп»). Скриншот в зуме на контрастном фоне.
3. **Классифицировать** (см. «Дерево решений»). Определи материал, структуру, слои глубины,
   форму текстуры, состояния, респонсив. Отдельно выпиши, что **нового** (нет в каталоге) —
   это надо разобрать и потом дописать в скилл.
4. **Архитектура до вёрстки.** Фон / декор-рама / эффект-FX → отдельные **компоненты**.
   Кнопка — горизонтальный **auto-layout**, размер **Hug**, при растяжении **3-слайс** (центр FILL).
   Состояния — варианты компонент-сета.
5. **Итерация 1 (быстро, можно реюз).** Переиспользуй внутренние компоненты/ассеты, собери,
   **сверь скриншотом в зуме**.
6. **Итерация 2 (сам).** Фон/декор векторами и слоями по методу (`references/vector-method.md`).
7. **Состояния + проверка** по чек-листу.

## Дерево решений: какой приём выбрать

Иди по признакам референса (можно комбинировать). Точные параметры — в `references/techniques.md`.

- **Форма с прямыми/скошенными краями, тянется по ширине** → 3-слайс (Left FIXED / Center FILL /
  Right FIXED) [T1].
- **Фон — единый РАСТР (image fill), кнопка тянется** → 9-slice растра через `scaleMode:CROP` +
  `imageTransform` с ОБЩИМИ долями швов [T2]. (Анти-паттерн: FILL или CROP с фикс-долей на всю
  ширину — узор растягивается.)
- **Неон/стекло, объём у кромок** → краево-сконцентрированные alpha-градиенты + 30% wash + dots [T3].
- **Гладкий глянец (пластик/гель)** → вертикальный градиент светлый верх→тёмный низ + верхний
  блик + нижняя фаска + INNER_SHADOW [T4].
- **«Пилюля» на тёмном UI, мягкий объём** → жёсткая цветная тень `blur:0` снизу + тонкий белый
  внутр. stroke + мягкая тень [T5].
- **Светящееся «ядро»** → радиальный градиент (яркий центр→тёмные края) + INNER_SHADOW [T6].
- **Органическая «энергия/молния/дым»** → РАСТР + `blendMode:LIGHTEN` + alpha-маска по форме
  (не вектор) [T7]; анимация — отдельный FX-компонент / GIF на hover (см. [T13]).
- **Металлическая рамка-оправа** → не одна обводка, а несколько концентрических rounded-rect с
  градиентами (внешний тёмный кант → металл → внутренний светлый кант) [T8]; рейлы — градиент-stroke.
- **Орнамент/филигрань по краям** → отдельный вектор-компонент, правый = левый зеркалом [T9]
  (рисовать по `references/vector-method.md`, не «ромбом»).
- **Блестящая поверхность** → блики: верхний глянец + диагональная засветка + INNER_SHADOW [T10].
- **«Выдавить» из фона + низ** → внешняя тёмная DROP_SHADOW (две: мягкая + плотная) + мягкий
  нижний блик (размытый эллипс, низкая opacity) [T11].
- **Лейбл должен «играть»** → градиентная заливка ТЕКСТА + жёсткая тень `0/1` [T12].
- **Нужны состояния** → один скелет, меняем свет/насыщенность/glow/наличие FX; компонент-сет [T14].
- **Анимация только на hover** → GIF-фон + текст слоем + прототип Default→Hover (`ON_HOVER`) [T13].
- **Текстура из «точек/стрелок»** → смотри ОБЩЕЕ: форма часто **эмерджентная** (точки складываются
  в стрелки), бывает несколько КРУПНЫХ направленных форм (слева→, справа←), и она НЕ тянется
  (фикс. сетка, `constraints:MIN`), база тянется отдельно [T15].
- **Острые углы с вогнутыми «вычерпнутыми» вырезами** (каменный/металл слэб) → boolean SUBTRACT:
  rect минус union из 4 эллипсов по центрам углов; у каждого Z-слоя свой размер резаков = ступенчатая глубина [T16].

## Объём/глубина — универсальная модель (важнее любого Txx)
Объём НЕ запоминается рецептом «как у клавиши/глянца» — он СОБИРАЕТСЯ по 5 рычагам.
Любой объёмный Txx ниже — частный случай этой модели с конкретной палитрой. Читай модель, рецепт — готовые числа.
1. **Свет один.** Выбери направление ОДИН раз (почти всегда сверху, часто сверху-слева). Дальше всё подчиняется:
   грани к свету — светлее, от света — темнее. Несогласованные блики/тени = «грязь».
2. **Форма поверхности задаёт градиент тела:**
   - приподнятая плоскость (клавиша/плитка/чип) → бевел по кромкам (OUTSET), тело почти ровное;
   - выпуклая (купол/пилюля/глянец) → сильный градиент тела свет→тень + specular-пятно;
   - вогнутая (поле ввода/трек/слот) → ИНВЕРСИЯ: тень сверху-внутрь, свет снизу-внутрь (INSET);
   - цилиндр (ствол/ролик) → градиент ПОПЕРЁК оси (свет с одной стороны).
3. **Стек слоёв = грамматика глубины (снизу вверх):** (a) тело: база+градиент по форме →
   (b) фаска: INNER_SHADOW светлый со стороны света + INNER_SHADOW тёмный с противоположной
   (OUTSET = свет/тень по РАЗНЫМ краям диагонально; INSET = оба внутрь, инверсия) →
   (c) блик: узкая яркая линия/пятно на грани к свету → (d) тень: DROP_SHADOW мягкая дальняя + плотная ближняя/контакт →
   (e) опц. окклюзия/wash в углублениях. **Минимум, чтобы не «плоско» = a+b+d.**
4. **Контраст = сила объёма.** Нужно больше объёма → РАЗВОДИ свет и тень сильнее (ярче блик, темнее тень),
   а не «другой оттенок». «Плоско» почти всегда = один градиент без кромок-фаски и без тени-отрыва.
5. **Материал = амплитуда и резкость переломов.** Металл — резкий свет→тень, узкие жёсткие блики, многостоповый градиент;
   пластик/гель — мягкие широкие переходы; матовое — слабый контраст. Тёплый свет / холодная-тёмная тень = «дорого».

**Само-проверка объёма (прогнать перед показом):** где источник света и согласованы ли ВСЕ блики/тени с ним?
есть ли a+b+d? это приподнято или вдавлено — фаска соответствует (OUTSET vs INSET)?
хватает ли контраста (сравни самый светлый блик с самой тёмной тенью, не «оттенок»)? читается ли материал по резкости переломов?

## Прочие переносимые принципы
- Негативное пространство — через `evenodd`/boolean/слой-снизу.
- Перекрас под тему = клон градиента + замена цветов стопов (тот же `gradientTransform`).
- Реюз > имитация: фон/декор/FX держим компонентами; перекрашиваем, не дублируем геометрию.

## Ведение каталога (чтобы не пух узкими рецептами)
Перед добавлением Txx спроси: это НОВЫЙ принцип/механика или ЧАСТНЫЙ СЛУЧАЙ существующего?
- Частный случай (та же модель, другая палитра/числа) → допиши ОДНОЙ строкой-примером под нужный приём, НЕ заводи новый Txx.
- Новый принцип (другая механика API или иная модель) → новый Txx.
Объёмные T3/T4/T5/T6/T8/T11/T16/T22 — все инстансы «модели объёма» выше (отличаются формой+материалом+inset/outset).

## Чек-лист сдачи (пройти ДО показа)
1. auto-layout? 2. размер Hug (высота паддингом, не Fixed)? 3. тянется по горизонтали (3-слайс/CROP)?
4. **фон/декор/FX вынесены в отдельный КОМПОНЕНТ и переиспользованы во ВСЕХ состояниях** (один imageHash/одна геометрия; состояния меняют только оверлеи/свет, не дублируют фон)? 5. глубина по МОДЕЛИ ОБЪЁМА (минимум a+b+d; свет один; контраст блик↔тень достаточен; outset/inset верно)?
6. **внешняя тень-отрыв и нижний блик на месте** (частый промах)? 7. **форма текстуры повторена**
(смотрел общее, не «горошек»)? 8. у нарезанного растра швы совпадают по пикселям?
9. шрифт по характеру оригинала? 10. нет лишнего чёрного контура? Любое «нет» — доделать.

## Грабли use_figma (критично)
Полный список и код-рецепты — `references/figma-recipes.md`. Самое важное:
- Ошибка в коде = **откат всей транзакции** → проверяй значения заранее.
- `use_figma` **не возвращает значения** → id новых нод узнавай через текст-слой + скриншот.
- `constraints` enum: `MIN|CENTER|MAX|STRETCH|SCALE` (НЕ TOP/BOTTOM).
- COMPONENT/COMPONENT_SET в `get_screenshot` часто рендерятся 1×1 → снимай **инстанс**, не компонент.
- GIF-фон: `upload_assets` POST из песочницы может блокироваться прокси → вшивай через
  `figma.createImage(bytes)` + ручной base64-декодер (встроенный `figma.base64Decode` строгий).
