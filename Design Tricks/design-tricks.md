# Design Tricks — рабочая инструкция по разбору и сборке кнопок

Назначение: **не воспроизводить конкретные кнопки, а решать новые** кнопки тем же способом.
Поэтому главное здесь — порядок действий и переносимые принципы; ниже — каталог приёмов с точными значениями и грабли API.

Источники-референсы: Dragonia `168693:130090`, Vegas Hero `168693:130089`, CasinoLab `168719:62762`, Cazeus `168765:124223`.

---

## Часть 1. ПОРЯДОК ДЕЙСТВИЙ (алгоритм для ЛЮБОЙ кнопки)

**Шаг 0. Понять задачу и доступ.** Есть ли Figma node-ссылка (можно читать слои/значения) или только картинка (только визуальный разбор). Финальный формат (Figma/код).

**Шаг 1. Разобрать референс по принципам (см. Часть 2).** Пройтись по чек-листу восприятия:
- Материал? (металл / стекло-неон / пластик-пилюля / энергия). 
- Откуда свет? (обычно сверху — где блик, где тень).
- Где глубина и как сделана? (краевые градиенты, INNER_SHADOW, внешняя/жёсткая тень, фаска).
- Есть ли негативное пространство? (вырезы, прорези, сквозь которые виден фон).
- Геометрия: скругление, скос, орнамент; тянется ли по горизонтали (значит 3-слайс).
- Слои: фон / текстура-эффект / рамка-декор / лейбл — что отдельно.
- Состояния: Default/Hover/Active/Disable/Loading — чем отличаются.
- Шрифт: кегль, наклон, цвет, тень.

**Шаг 2. Извлечь факты (не угадывать).**
- Если есть ссылка: `get_metadata` (структура) → `get_design_context` (композиция, ассеты) → `get_variable_defs` (токены) → дамп через `use_figma` точных `fills`/`opacity`/`blendMode`/`effects`/`strokes`/`cornerRadius` и `vectorPaths`. Вектора/картинки — `download_assets` (svg/png).
- Если только картинка: снять палитру, радиусы, тени визуально; значения подбирать и сверять скриншотом.

**Шаг 3. Классифицировать приёмы.** Какие из каталога (Часть 4) подходят: 3-слайс? alpha-глубина? жёсткая тень-объём? радиал-ядро? текстура+blend? рама из орнаментов? негативное пространство? Отдельно выписать, что **новое** (чего нет в каталоге) — это и есть то, что надо изучить.

**Шаг 4. Спроектировать архитектуру (до вёрстки).**
- Фон / декор-рама / эффект-FX — в **отдельные компоненты**.
- Кнопка: горизонтальный **auto-layout**, размер **Hug**, при растяжении — **3-слайс** (центр FILL).
- Состояния — варианты компонент-сета.

**Шаг 5. Итерация 1 — быстро, можно с реюзом.** Переиспользовать готовые внутренние компоненты/ассеты, собрать композицию, **сверить скриншотом в зуме** с референсом.

**Шаг 6. Итерация 2 — сделать самому.** Фон/декор векторами и слоями по методу (Часть 5): реальные пути → кривые → негативное пространство → слои глубины → материал-градиент. Перекрас под тему — клоном градиента.

**Шаг 7. Состояния + проверка.** Собрать состояния (менять только свет/насыщенность/glow/FX). Прогнать чек-лист (Часть 3 / правила). Где «нет» — доделать. Высокий риск — проверить субагентом/дизайн-скиллом.

> Правило извлечения: **точные значения берём из файла, а не из памяти.** «Скопировать и понять» лучше, чем «придумать на глаз».

## Часть 2. ПЕРЕНОСИМЫЕ ПРИНЦИПЫ (применимы к любой кнопке)

- **Глубина — это всегда СТЕК слоёв, не один fill.** Порядок: база → краевые градиенты с альфой (светлее у верхней кромки, темнее у нижней) → `INNER_SHADOW` (белый сверху = глянец, тёмный снизу = вдавленность) → внешняя тень (подъём/толщина). Жёсткая тень без размытия = физическая грань/толщина.
- **Единый источник света.** Все блики, фаски и тени согласованы (по умолчанию свет сверху). Верх светлее, низ темнее — почти всегда.
- **Негативное пространство.** Вырезы/прорези в форме (сквозь которые виден фон или энергия) дают дорогую детализацию и «лёгкость» массивной формы. Делать через `fill-rule=evenodd` / boolean-вычитание, либо отдельным слоем под фигурой. Применять в орнаментах, рамках, иконках, заглушках.
- **ФОРМА важна, и смотреть надо ОБЩЕЕ, а не частное.** Форма часто **эмерджентная**: примитив — простая **точка**, а макро-форму дают их группировка И КОМПОЗИЦИЯ. Важно увидеть КОМПОЗИЦИЮ целиком, а не только примитив:
  - westace-текстура = НЕ равномерное поле точек, а **несколько БОЛЬШИХ стрелок** из точек, расположенных **симметрично и направленно**: слева стрелки смотрят вправо (→), справа — влево (←) (к центру). Равномерный «горошек» = брак («кнопка как будто болеет»).
  - Алгоритм: (1) увидеть макро-композицию (сколько форм, где, куда направлены, симметрия); (2) увидеть силуэт каждой формы (стрелка); (3) увидеть примитив (точка) и плотность. Воспроизводить все три уровня, а не один.
  - Мои повторные промахи: ромб вместо филиграни; стрелка-глиф вместо точек; равномерное поле вместо нескольких крупных направленных стрелок.
- **Респонсив текстуры: она НЕ тянется.** Узор (стрелки/точки) при растяжении кнопки должен сохранять размер и просто повторяться/открываться, а не масштабироваться. Базовый слой (гладкий градиент/рамка) тянуть можно, узор — нет.
- **Контраст и напряжение.** Крупный, очень тёмный (или подчёркнуто светлый) лейбл на ярком фоне; наклон шрифта рифмуется с наклонными линиями фона. Шрифт — часть приёма, не «подпись».
- **Материал решает приём.** Металл = многостоповый градиент с РЕЗКИМ свет→тень переломом (бликовая грань). Стекло/неон = краевые alpha-градиенты + свечение. Пластик-пилюля = мягкий вертикальный градиент + тень-объём. Энергия/дым = растр + blend (lighten) + маска (не вектор).
- **Цветовые пары.** Тёплый металл × холодная энергия (комплементарность): золото/бронза + синий читаются «дорого». Перекрас металла — клон градиента + замена цветов стопов (тот же `gradientTransform`).
- **Респонсив = структура.** Тянущаяся кнопка = 3-слайс (углы FIXED, центр FILL) + Hug-сайзинг.
- **Переиспользование = компоненты.** Фон/декор/FX — отдельные компоненты; состояния — варианты. Растровые фоны в компоненте экономят память (один imageHash).

## Часть 3. Жёсткие правила (чек-лист сдачи)

1. Только **auto-layout**, без свободных шейпов с ручными координатами.
2. Размер — **Hug** (высоту держать паддингом, не Fixed).
3. Тянущаяся кнопка — **3-слайс** (центр FILL).
4. Фон/декор/FX — **компоненты**, перекрас через клон градиента.
5. Глубина — **слоями** (alpha-градиенты у кромок + INNER_SHADOW + wash + тени), не одним градиентом.
6. **Негативное пространство** там, где в референсе есть вырезы — через evenodd/boolean/слой-снизу.
7. Шрифт — по характеру оригинала (крупный/наклонный/тёмный/серифный).
8. Металл — **многостоповый** градиент с резким переломом; единый свет.
9. Без декоративного чёрного `stroke` (если его нет в оригинале); рамка-рейл часто — **градиент-stroke**.
10. У нарезанного растра **швы совпадают по пикселям** (CROP+imageTransform, общие доли).
> Перед сдачей пройти пункты 1–10; любое «нет» — доделать.

## Часть 4. Каталог приёмов (с точными значениями)

### 4.1 Масштабируемый 3-слайс
Горизонтальный auto-layout; края `layoutSizingHorizontal="FIXED"`, центр `"FILL"`, высота `"FILL"`. Фон-3-слайс — абсолютный, `constraints {STRETCH,STRETCH}`, под лейблом.

### 4.2 9-slice растра с совпадением швов
Каждый слайс — прямоугольник, `fills=[{type:"IMAGE",scaleMode:"CROP",imageHash,imageTransform:[[cw,0,L],[0,1,0]]}]`. Соседние слайсы делят **одинаковую долю-границу** (пример cap=0.178 → left `L=0,cw=.178`; center `L=.178,cw=.644`; right `L=.822,cw=.178`). Центр FILL, капы FIXED → центр тянется, швы не уезжают.

### 4.3 Глубина alpha-градиентами + wash (неон/стекло, Vegas Hero green)
Слои Center: база `#a9f29d` → wash `#a9f29d @0.30` → тёмная фаска градиент `#121723` альфа `[1@0, 0@0.47, 1@1]` (+INNER_SHADOW) → точки (тайл-PNG) → неон градиент `#6fea5c` альфа `[1@0, 0@0.19, 0@0.76, 1@1]`. Токены: `#6fea5c / #a9f29d / #c5f7be`.

### 4.4 Глянец (гладкая поверхность)
Вертикальный градиент светлый верх→тёмный низ + верхний блик + нижняя фаска + INNER_SHADOW. Пример (зелёная): `#e9ffd0→#b9f78d→#74e646→#47b82f→#2c8a22`; блик белый `0.75→0` сверху; фаска тёмная `0→0.5` снизу.

### 4.5 Объём «пилюли» (CasinoLab)
Pill (r=½H), мягкий градиент, тонкий белый `stroke #ffffff@0.20 w1 INSIDE` сверху, **жёсткая тень снизу** `DROP_SHADOW off(0,5) blur0` цветом темнее заливки (+ опц. мягкая `blur15 a0.4`). Темы: orange fill `#ffe11f→#ff8533` hard `#e36d1f`; blue `#cddcfb→#9bbaf8` hard `#588cf3`; purple `…→#cbbcff` hard `#a88fff`; ghost `#ffffff→#eef3fe` hard `#dee8fd`.

### 4.6 Радиальное ядро свечения (Cazeus BG)
Радиал яркий центр→тёмные края + INNER_SHADOW + тонкий металл-`stroke`. Значения: radial `#1b6dff→#1757ce→#14419d→#102a6c→#0e1f54→#0c143b`; border `#ffb800` (бронза `#cd7f32`); inner `inset 0 4 4 #919bf3` + `inset 0 -3 6 rgba(102,157,255,.6)`. Радиал делать через `createNodeFromSvg` (`<radialGradient cx=50% cy=44% r=62%>`).

### 4.7 Энергетическая текстура: растр + blend + маска
Растр накладывать с `mix-blend-mode:LIGHTEN`, обрезать **alpha-маской** по форме, резать на Left/Middle/Right для масштаба. Анимация — компонент с 2 кадрами.

### 4.8 Рама = орнамент-капы + градиент-рейлы
Орнамент — отдельный компонент-вектор, правый = левый зеркалом (`relativeTransform=[[-1,0,x+w],[0,1,0]]`). Верх/низ — рамка с обводкой только сверху/снизу (`strokeTopWeight/strokeBottomWeight`, бока 0), **обводка-градиент** `#ffb800@0→#5b2c15@1`.

### 4.10 Градиентная заливка ТЕКСТА (приём был и раньше — теперь записан)
> Процесс-урок: градиент-текст встречался и прежде, но я его НЕ записывал → выдал за «находку». Записывать даже знакомое.
- **Суть:** лейбл залит вертикальным градиентом (light верх → акцент низ) + жёсткая тень `0/1/0 #000` (blur 0) = эмбосс.
- **Значения:** `#ffffff` (0–40%) → `#b1ff9a`; Fira Sans Condensed ExtraBold, uppercase, tracking ~2%.
- **Figma:** у TEXT-ноды `fills=[GRADIENT_LINEAR]`, `effects=[DROP_SHADOW off(0,1) blur0 #000]`.

### 4.11 Респонсивная текстура, которая НЕ тянется (настоящий приём westace)
- **Задача:** при растяжении кнопки узор-стрелки сохраняют размер, их становится больше (а не растягиваются).
- **Как в оригинале:** фон — один растр, слайсы — прямоугольники со `scaleMode:"CROP"` (`scalingFactor` задаёт зум). CROP держит изображение в **постоянном масштабе** и при росте слайса **открывает больше**, а не масштабирует. Торцы фикс. ширины (фикс. кроп), центр растёт (открывает середину). Узкий центр (29px) + широкие торцы (113/115).
- **Альтернатива (TILE):** `scaleMode:"TILE"` + `scalingFactor` — узор повторяется при постоянном размере; самый чистый способ для бесконечного паттерна.
- **Векторный способ (мой реюз-фри):** узор — сетка фикс-размерных вектор-стрелок в клип-слое с `constraints {horizontal:"MIN"}` (не SCALE); при росте контейнера стрелки не масштабируются, открывается больше. Базовый слой (градиент/рамка) при этом тянется отдельно.
- **Ошибка, которой избегать:** `scaleMode FILL` или `CROP` с `imageTransform`, натягивающим фикс-долю на всю ширину → узор РАСТЯГИВАЕТСЯ. Так делать нельзя.

### 4.13 Многослойная металлическая рамка (несколько градиентов)
- **Суть:** золотая рамка — не одна обводка, а **стопка концентрических rounded-rect'ов**, у каждого свой градиент → объёмная фаска.
- **Слои (снаружи внутрь):** внешний тёмный кант `#7a5618→#2e1d06` → основной металл `#ffe98a→#f0c040→#a9760f` (inset ~1.5) → внутренний светлый кант `#fff6cf→#caa033` (inset ~3.5) → сам цветной панель (inset ~5.5) сверху. Все слои `constraints STRETCH` (инсеты постоянны при ресайзе).
- **Где:** премиальные «оправы» кнопок/карточек; даёт металл лучше, чем один gradient-stroke.

### 4.14 Блики на фоне (glints)
- **Суть:** на основном градиенте — несколько источников света: (1) **верхний глянец** — широкая полоса белого `0.55→0` у верха; (2) **диагональная засветка** — повёрнутый (~20°) узкий прямоугольник с белым `0→0.28→0` (полупрозрачная «искра»); (3) `INNER_SHADOW` белый сверху (glint-кромка) + тёмный снизу (глубина).
- **Где:** глянцевый пластик/стекло/гель-кнопки.

### 4.12 9-slice из ОДНОЙ картинки (память)
Все слайсы ссылаются на **один** `imageHash` (разные доли). Один растр на кнопку — минимум памяти; швы по общим долям (см. 4.2).

### 4.9 Состояния
Один скелет, меняем свет/насыщенность/glow/наличие FX. Hover — ярче + внешний glow; Active — темнее + «вдавленный» INNER_SHADOW + FX ~50%; Disable — обесцвечено, FX off, `opacity ~0.65`. Держать вариантами компонент-сета.

## Часть 5. Метод рисования векторов

1. **Брать реальный путь, не угадывать:** `node.vectorPaths[i].data` → копировать 1:1 в новый вектор; из чужого файла — `download_assets` (svg).
2. **Кривые Безье (`C`), не полилинии;** рисовать половину — **зеркалить**.
3. **Негативное пространство** — `fill-rule evenodd`/boolean или слой под фигурой.
4. **Материал** — многостоповый градиент с резким переломом ~0.5 (пример золота: `#ffe701@0,#ffc757@.34,#fde1a9@.49,#6e531c@.51,#9b8455@.58,#ac7e22@.67,#573723@.88,#5b2c18@1`).
5. **Итерация:** скриншот фигуры в зуме → сверка → правка точек (быстрые правки — в SVG-песочнице).

## Часть 6. Грабли use_figma (Plugin API)

- `constraints` enum: `MIN|CENTER|MAX|STRETCH|SCALE` (НЕ `TOP/BOTTOM`).
- Ошибка в коде = **откат всей транзакции**; проверять значения заранее.
- `use_figma` **не возвращает значения** — id узнаю через текст-слой + скриншот.
- Абс. фон в auto-layout: `layoutPositioning="ABSOLUTE"` + `constraints STRETCH` + `resize`.
- Зеркало: `relativeTransform=[[-1,0,x+w],[0,1,0]]`. Градиент-stroke: `strokes=[GRADIENT_*]` + `strokeTop/BottomWeight`.
- `combineAsVariants(comps,parent)` — на одной странице (сперва `parent.appendChild`); имена `"State=…"`. `createComponentFromNode(frame)` — фрейм→компонент.
- `cornerRadius`/`strokeWeight` бывают `figma.mixed` (symbol) — приводить безопасно.
- Текстуры/гриды — `createNodeFromSvg`.
