Файл-копилка приёмов вёрстки UI-компонентов, разобранных по реальным макетам.

Формат записи на каждый приём: **что видно → как устроено в слоях → как повторить → где применимо**.

## ⛔ ЖЁСТКИЕ ПРАВИЛА (соблюдать всегда, проверять перед сдачей)

1. **ЗАПРЕТ на «просто набор шейпов».** Кнопка/контейнер НЕ собирается из свободно лежащих векторов с ручными координатами. Всегда **auto-layout** (frame с layoutMode), чтобы элемент респонсился. Ручной набор шейпов = брак, переделывать.
2. **3-слайс на лейаутах = обязателен для растягиваемых кнопок.** Left (hug/fix) + Center (`layoutGrow=1` / FILL) + Right (hug/fix) в горизонтальном auto-layout. Углы фиксированы, центр тянется под подпись. (Правило было — теперь применять, а не только знать.)
3. **Фон — отдельный КОМПОНЕНТ.** Фоновую графику кнопки выносить в переиспользуемый компонент (его инстансы вставляются в разные кнопки). Особенно когда внутри растровые изображения — **это экономит память** (один image-fill переиспользуется, а не копируется). Реюз внутреннего фон-компонента всегда предпочтительнее имитации вектором.
4. **Шрифт лейбла — часть приёма, а не «подпись».** Если в оригинале крупный наклонный очень тёмный шрифт — повторять именно так: большой кегль, italic/наклон, очень тёмный цвет. Это даёт контраст и «напряжение», а наклон шрифта рифмуется с наклонными линиями фона. Мелкий прямой шрифт «не того цвета» = брак.
5. **Глубину строить полигонами с разной прозрачностью.** Каждый слайс — стопка полигонов с разной opacity; при наложении дают интересные переливы/объём. Не один fill.
6. **Текстуру давать растром.** В центральную часть класть PNG с точками/паттерном для «материальности» поверхности.
7. **Глубину дают КРАЕВО-сконцентрированные градиенты с альфой, а не ровный градиент.** Объём строится так: плоская светлая база + поверх неё градиенты, у которых альфа = 1 у верх/низ кромок и = 0 в середине (тёмный для фаски, яркий для неона). Плюс полупрозрачный wash (~30%). Ровный top→bottom градиент = плоско и непохоже. Это конкретизация правила №5.
8. **После нарезки растра — швы обязаны совпадать по пикселям.** Границы слайсов (cap↔center) должны попадать в ОДНИ И ТЕ ЖЕ пиксели источника. Иначе при ресайзе центр «уезжает». Реализация в Figma: `scaleMode:"CROP"` + `imageTransform:[[cw,0,L],[0,1,0]]`, где у соседних слайсов общая граница задаётся одинаковой долей `L`. Тогда тянется только средний кроп, стык не дрейфует.
9. **Проверка перед сдачей:** есть ли auto-layout? тянется ли по горизонтали? фон — компонент/инстанс? шрифт по характеру оригинала (крупный, наклонный, тёмный)? глубина через alpha-градиенты у кромок + wash? текстура на месте? у нарезанного растра швы совпадают по пикселям? Если хоть одно «нет» — доделать.

## Приём №1 — Масштабируемая кнопка из 3 слайсов (Left / Center / Right)

**Где:** Vegas Hero → `buttons-parts` (`168693:127524`), компоненты `Property 1 = Left/Center/Right`. На этой же системе собраны и зелёные «срезанные» кнопки, и золотые CTA.

**Что видно.** Кнопка со скошенными (шевронными) краями: острые уголки слева и справа, прямая середина. При любой длине подписи углы НЕ растягиваются и не искажаются — тянется только середина.

**Как устроено в слоях.**
- Контейнер кнопки — горизонтальный auto-layout (flex-row): `Left` (фикс. ширина ~32px) + `Center` (`flex: 1 0 0`, тянется) + `Right` (фикс. ~32px).
- Каждый слайс `overflow: clip` и состоит из СТОПКИ полигонов (отдельные слои), а не одной заливки:
  - `Polygon` базовый — `inset: 0` (полный размер) → это внешний контур/свечение, самый тёмный/насыщенный.
  - `Polygon` заливки — `inset: 4.76%` сверху и снизу. 4.76% от 42px ≈ 2px → это толщина бордера. Внутренняя заливка «вложена» на 2px внутрь контура.
  - Дополнительные полигоны — внутренний хайлайт/градиент и тонкая обводка.
  - В центре ещё слой `Poits` — точечная/пиксельная текстура (декор внутри кнопки).
- Диагональная грань делается так: берётся вертикальный полигон и поворачивается на 90° с растяжением через container-query единицы (`h-[100cqw] rotate-90 w-[100cqh]`). Это позволяет одному и тому же шеврону тянуться по высоте слайса без ручного пересчёта.
- Right-слайс — это тот же Left, но `rotate-180` (зеркало), экономия на ассетах.

**Почему это «трюк».** Классический 9-slice/3-slice: дорогую, сложную по форме кнопку рисуют один раз как набор кусочков, а потом собирают auto-layout'ом. Один компонент покрывает все размеры (L/M/S/XS/XXS) и все подписи — середина просто тянется. Углы пиксель-в-пиксель одинаковые на всех размерах.

**Как повторить (нативно в Figma, без ассетов).**
1. Frame с горизонтальным auto-layout, gap 0, align center.
2. Left = vector-шеврон (полигон), фикс. ширина. Right = дубликат Left, flip horizontal.
3. Center = frame, `fill container` по ширине, прямоугольник с заливкой.
4. На каждый кусок — стопка: тёмный контур (низ) → основная заливка (inset 2px) → линейный градиент-хайлайт сверху (overlay/screen, низкая прозрачность).
5. Сверху — текстовый слой подписи поверх всех слайсов (absolute / в отдельном auto-layout-ряду).

**Как повторить (если форма слишком сложная).** Переиспользовать исходные PNG-полигоны слайсов из компонента и собрать их тем же auto-layout'ом — края останутся «родными».

**Применимо к:** игровые/гейминг-кнопки, кнопки с нестандартной геометрией (срезы, скосы, «билеты»), любые кнопки, где важно сохранить угол при растяжении.

## Приём №2 — Многослойная заливка вместо одного fill (depth-стек)

**Где:** все полигоны слайсов Vegas Hero (`Polygon 1–4` внутри `Center` `168693:127504`).

**Что видно.** Кнопка выглядит «объёмной»: яркий верх, более тёмный низ, светящаяся рамка, лёгкая внутренняя текстура. Это не один градиент.

**Как устроено.** На одну форму кладётся 3–4 совпадающих по силуэту слоя:
1. **Контур/свечение** — самый нижний, полный размер, насыщенный/тёмный цвет (играет роль бордера и outer glow).
2. **Основная заливка** — вложена на ~2px (`inset 4.76%`), вертикальный линейный градиент (светлее сверху).
3. **Хайлайт** — узкий градиент у верхней грани, режим наложения screen/overlay, низкая непрозрачность.
4. **Текстура** (`Poits`) — мелкие точки/пиксели для «материала».

**Почему это «трюк».** Глубина и «сочность» собираются из плоских слоёв с разным inset и blend-режимом, а не одним сложным эффектом. Легко перекрашивать под состояние (Hover/Pressed/Disabled) — меняешь цвета слоёв, геометрия не трогается.

**Как повторить.** Дублируешь форму 3–4 раза, каждой даёшь свой fill/inset/blend. Группируешь. Для состояний — варианты компонента с теми же слоями, но другой палитрой.

**Применимо к:** любые «премиальные»/игровые поверхности — кнопки, бейджи, карточки, чипы.

## Приём №3 — Матрица вариантов: Type × Size × State

**Где:** `button` component set Vegas Hero (`168693:127617`).

**Что видно.** Один компонент-сет покрывает: Type = Primary / Secondary / Tertiary / Quaternary / Fifth; Size = L / M / S / XS / XXS; State = Default / Hover / Click / Disable / Loading.

**Как устроено.** Component set с тремя свойствами (variant properties). Размеры заданы фикс. высотами: L 68px, M 56px, S 48px, XS/XXS 36px. Все варианты лежат сеткой, имена в формате `Type=…, Size=…, State=…`.

**Почему это «трюк».** Полная матрица состояний в одном компоненте → консистентность и быстрый свитч в дизайне/коде. Loading и Disabled заведены как полноценные состояния, а не «забыты».

**Применимо к:** любая дизайн-система кнопок/инпутов/чипов.

## Эксперимент №3 (round 2, 2026-06-12) — фрейм `168709:2`, рядом с round 1

Новые варианты ставлю в отдельный фрейм СПРАВА, прошлые не удаляю. Нумерую.

### [3] Золото на 3-слайс (cap-left / center-stretch / cap-right)
**Задача:** золото — единый растр, при растяжении кропилась рамка. Решение — нарезать растр 3-слайсом БЕЗ редактирования картинки, через clip-окна:
- `cap-left` = frame `clipsContent=true`, внутри прямоугольник со ВСЕЙ золотой картинкой (image fill, `scaleMode FILL`) в натуральных пропорциях (≈360×92), позиция `x=0` → окно показывает левый кованый угол.
- `cap-right` = то же, но внутренний прямоугольник сдвинут `x = -(IMG_W - capW)` → окно показывает правый угол.
- `center` = frame с тем же image-fill `FILL`, `clipsContent`, в auto-layout `FILL` → тянется (середина фактуры).
- Сборка: горизонтальный auto-layout, капы `FIXED`, центр `FILL`; bg абсолютный со `STRETCH`.
**Результат:** кованые углы чёткие на любой ширине (`PLAY` и длинный `GET BONUS`), рамка не кропится. Тот же imageHash переиспользуется во всех трёх слайсах → память не растёт.

### [4] Зелёная — фон собран САМОСТОЯТЕЛЬНО (без реюза чужого компонента)
Создал свой компонент `Green BG / self-made (component)` — 3-слайс, каждый слайс из слоёв:
- **Капы** (триангл-вектор `createVector` с `vectorPaths`): нижний слой — тёмная триангл-рамка `#2f7d2a`; сверху — триангл-заливка с инсетом 3px, вертикальный градиент `#c5f7be→#46a828`. Зеркалю для левого/правого.
- **Центр** (стопка с разной прозрачностью — это и есть приём №5):
  1. `dark border` — прямоугольник `#2f7d2a` на всю высоту (рамка верх/низ).
  2. `fill` — прямоугольник инсет 3px, вертикальный lime-градиент `#c5f7be→#6fea5c→#46a828`.
  3. `white sheen` — верхняя полоса, градиент белый `0.45→0` (полупрозрачный блик).
  4. `bottom shade` — нижняя полоса, градиент тёмный `0→0.5`.
  5. `dotted texture` — **сам сгенерировал SVG-сетку кружков** через `figma.createNodeFromSvg(...)`, `opacity 0.20`, прижата влево и кропится центром (фиксированная плотность).
- Сборка кнопки: те же правила — auto-layout, инстанс самодельного фон-компонента абсолютным `STRETCH`, лейбл наклонный тёмный.
**Результат:** выглядит как оригинал по структуре, респонсив работает. **Я умею собирать такой фон сам, не только реюзить.**

### 📌 Метод реюза vs самосбор (запомнить)
- **Реюз готового компонента-фона — самый действенный способ, КОГДА есть что переиспользовать.** Берём инстансы существующих слайсов (Left/Center/Right), собираем auto-layout'ом — получаем «бесплатно» полигоны с прозрачностью и текстуру, и память не растёт.
- **Но обязан уметь и сам:** триангл/полигон-векторы (`createVector`+`vectorPaths`), стопка слоёв с разной `opacity` и `constraints` (`STRETCH`/`MIN`/`MAX`/`SCALE`), текстура через `createNodeFromSvg`, всё в горизонтальный auto-layout с центром `FILL`. Эти два пути взаимозаменяемы; выбор — есть ли готовый ассет.

### ⚙️ Тех-заметки use_figma (грабли, запомнить)
- `constraints.vertical` принимает только `MIN | CENTER | MAX | STRETCH | SCALE` (НЕ `TOP/BOTTOM`). Горизонталь — те же.
- Растягиваемый фон в auto-layout-кнопке: `bg.layoutPositioning="ABSOLUTE"` + `constraints STRETCH` + `resize(parent.w,parent.h)`; внутри bg — свой горизонтальный auto-layout, центр `layoutSizingHorizontal="FILL"`, капы `FIXED`.
- Любая ошибка в коде use_figma откатывает ВСЮ транзакцию (ничего не создаётся) — проверять валидность значений заранее.
- use_figma не возвращает значения наружу; id новых нод узнаю, записав их в текстовый слой и сняв скриншот.

## Приём №5 — Объём через жёсткую цветную тень + асимметричные толщины (CasinoLab)

**Где:** CasinoLab `168719:62762` — простые pill-кнопки (Primary/Secondary/Tertiary/Fourth × размеры × состояния). Реплики — фрейм `168724:564` [8].

**Что видно.** Гладкие «пилюли» (полное скругление) с лёгким объёмом: будто у кнопки есть физическая толщина снизу, и она чуть приподнята над фоном.

**Как устроено (реальные значения Primary L):**
- Форма: `cornerRadius` = половина высоты (pill). Заливка — мягкий вертикальный градиент (светлый верх → насыщенный низ), напр. orange `#ffe11f → #ff8533`.
- **Объём = ДВЕ тени:**
  1. **Жёсткая цветная тень БЕЗ размытия** — `DROP_SHADOW`, `offset(0, 5)`, `blur 0`, `spread 0`, цвет = более тёмный/насыщенный тон кнопки (orange `#e36d1f`, blue `#588cf3`, purple `#a88fff`, ghost `#dee8fd`). Даёт сплошную «ступеньку» снизу = толщина корпуса.
  2. **Мягкая тень** (не у всех) — `blur ~15`, `alpha ~0.4` — ambient-подъём над фоном.
- **Разные толщины «фрейма»:** сверху — тонкая светлая внутренняя обводка `stroke #ffffff @0.20, w=1, align INSIDE` (блик-кромка), снизу — жёсткая тень 5px (тёмная кромка). Асимметрия тонкий-верх / толстый-низ и читается как объём/фаска.

**Почему это «трюк».** Объём без вектора и без растра — только тень с `blur:0` (работает как сплошная грань) + 1px светлый внутренний штрих. Дёшево, чисто масштабируется, легко перекрашивается под тему/состояние (меняешь цвет тени и заливки).

**Как повторить.** Frame (auto-layout, pill-радиус) → градиент-заливка → `strokes`=белый 0.20 w=1 INSIDE → `effects`=[мягкая тень blur16 a0.4 ; жёсткая тень off(0,5) blur0 a1, цвет темнее заливки]. Лейбл болдовый. (Сделано в [8] для 4 тем — совпало.)

**Применимо к:** обычные UI-кнопки, чипы, тогглы, бейджи — везде, где нужен «мягкий» объём без тяжёлой графики.
