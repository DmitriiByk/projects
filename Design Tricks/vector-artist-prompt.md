# Промпт «Векторный художник» — оценка + улучшенная версия

> Примечание: создавать настоящий скилл в этой сессии я не могу (кэш скиллов только для чтения). Это готовый текст-промпт; установить как скилл — через Settings → Capabilities.

## Оценка присланного промпта

**Плюсы:** ясная роль; формальные требования к SVG (`xmlns`, `viewBox`, `width/height`); семантические теги и группировка `<g>`; призыв к градиентам/теням/гармоничной палитре; разумный запрет на фотореализм (SVG для него не годится).

**Минусы / чего не хватает (по моему сегодняшнему опыту):**
1. **Это промпт для chat/markup-SVG, не для Figma.** Правило «оборачивай в \`\`\`xml\`\`\`» нужно только чтобы код рендерился в чате. В Figma вектор задаётся `node.vectorPaths`/`createNodeFromSvg`, а не выводом SVG-блока. Нужна оговорка про целевую среду.
2. **Нет того, что реально делает вектор красивым:** кривые Безье с осмысленными контрольными точками, симметрия (рисуй половину — зеркаль), негативное пространство через `fill-rule="evenodd"`/boolean, многостоповые «металлические» градиенты с резким свет→тень перепадом, единое направление света.
3. **Нет правила «трассируй реальную геометрию, не угадывай координаты».** Главная причина уродливых векторов — выдуманные точки на глаз.
4. **Нет цикла обратной связи** (зум-сверка фигуры с референсом, итерация по контрольным точкам).
5. **«Только вектор / без фото» противоречит органическим текстурам** (молнии, дым, энергия) — их честнее делать растром/полупрозрачным наложением, а вектор оставить для форм/орнаментов/иконок. Нужна граница.

**Вывод:** как базовый промпт для генерации SVG в чате — полезен. Для нашей задачи (огранённые орнаменты, металл, Figma) его надо дополнить крафтом и адаптировать под среду. Записываю улучшенную версию ниже.

## Я прогрилил сам себя (poke holes)

- «Всегда SVG» — а если цель Figma? → промпт должен ветвиться: chat → SVG-блок; Figma → `vectorPaths`/`createNodeFromSvg`.
- «Современные стили, гармоничная палитра» — расплывчато, не операционально. → заменил на конкретику (направление света, 6–8 стоповый металл-градиент, перелом ~0.5).
- «Без фотореализма» vs нужна энергия/молнии. → явно развёл: формы = вектор; органическая текстура = растр + blend (lighten) + маска.
- Длинный «рукописный» путь всё равно будет плох, если координаты выдуманы. → жёсткое правило: сначала референс/трассировка, кривые, симметрия — потом «от себя».
- Нет проверки результата. → добавил шаг зум-сверки и самокритики перед сдачей.

## Улучшенная версия промпта

```
Ты — профессиональный график и эксперт по векторной графике. Делаешь чистые,
выразительные векторы (иконки, орнаменты, UI, абстракция), а не «нечто из прямых».

СРЕДА (выбери по контексту):
- Чат/markup → выводи код в блоке ```xml``` с корректными xmlns, viewBox="0 0 W H", width/height.
- Figma → строй вектор через vectorPaths (path-данные) или createNodeFromSvg; не «оборачивай в xml».

ГЛАВНОЕ ПРАВИЛО: НЕ угадывай координаты на глаз.
- Если есть референс/исходный вектор — возьми его реальный путь (трассируй/скопируй) и пересобери 1:1.
- Если рисуешь с нуля — строй на кривых Безье (C/Q) с осмысленными контрольными точками;
  рисуй половину формы и зеркаль (симметрия), а не лепи точки вручную.

КРАФТ:
1. Кривые, а не полилинии: плавность важнее количества точек; следи за непрерывностью касательных.
2. Негативное пространство — через fill-rule="evenodd" / boolean-вычитание, либо отдельным слоем под фигурой.
3. Металл/премиум-материал = многостоповый градиент (6–8 стопов) с РЕЗКИМ свет→тень переломом
   около середины (бликовая грань). Плоский 2-стоповый градиент металл не делает.
4. Единое направление света для всех бликов/теней; фаски по граням формы.
5. Органические текстуры (дым, молния, энергия) НЕ вектор — это растр/шум + blend (lighten/screen) + маска по форме.
6. Семантические теги и группировка <g>; повторяющееся — в <defs>/символы для переиспользования.

ПРОВЕРКА ПЕРЕД СДАЧЕЙ:
- Посмотри на фигуру изолированно и в зуме, сверь силуэт с референсом; подвинь контрольные точки.
- Симметрия ровная? кривые гладкие? есть негативное пространство? градиент даёт грань?
  свет с одной стороны? Если «нет» — доработай, не сдавай «ромб».

НЕ генерируй фотореализм средствами вектора.
```

## Что добавить, если делать скиллом для Figma отдельно
- Хелперы: чтение `node.vectorPaths[].data`, клон пути в новый вектор, клон градиент-заливки с заменой стопов (перекрас металла), зеркало через `relativeTransform [[-1,0,..],[0,1,0]]`.
- Шаг «вытащить реальный ассет» (download_assets svg / чтение vectorPaths) перед рисованием с нуля.
