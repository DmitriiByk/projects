# Как рисовать красивые векторы (а не «ромб»)

Причина уродливых векторов — выдуманные координаты из прямых линий. Правильно:

1. **Брать реальную геометрию, не угадывать.** Путь читается из ноды: `node.vectorPaths[i].data`.
   Копируй 1:1 в новый вектор: `v.vectorPaths=[{windingRule:p.windingRule,data:p.data}]`.
   Из чужого файла — `download_assets` (format `svg`).
2. **Кубические кривые Безье (`C`), не полилинии (`L`).** Красота = плавные кривые с осмысленными
   контрольными точками + симметрия: рисуй половину формы и **зеркаль**
   (`relativeTransform=[[-1,0,x+w],[0,1,0]]`).
3. **Негативное пространство** — дырка в самом пути (`fill-rule evenodd` / boolean-вычитание),
   либо отдельный слой ПОД фигурой (в Cazeus синий `#1b6dff` проступает сквозь золото).
4. **Металл = многостоповый градиент с резким переломом** ~0.5 (бликовая грань). Пример золота:
   `#ffe701@0, #ffc757@.34, #fde1a9@.49, #6e531c@.51, #9b8455@.58, #ac7e22@.67, #573723@.88, #5b2c18@1`.
   Перекрас под бронзу/серебро: те же позиции стопов, другие цвета, тот же `gradientTransform`.
5. **Смотреть ОБЩЕЕ → силуэт → примитив.** Сначала макро-композиция (что складывается, сколько форм,
   направление, симметрия), потом силуэт, потом примитив. Не подменяй точку макро-формой.
6. **Итерация с обратной связью по ФИГУРЕ:** скриншот изолированно и в зуме → сверка с референсом →
   правка контрольных точек. Быстрые правки пути удобно крутить в SVG-песочнице, готовое — в Figma
   (`figma.createNodeFromSvg(svgString)` для импорта вектора/сетки фигур).

## Анти-паттерны (мои реальные промахи)
- Ромб из прямых вместо филиграни на кривых.
- Каждый элемент текстуры сделан «стрелкой» вместо точек, складывающихся в стрелку.
- Равномерный «горошек» вместо нескольких крупных направленных стрелок.
- Плоский 2-стоповый градиент на металле (нет бликовой грани).
- Лишний декоративный чёрный `stroke` на вектор-форме (форму задаёт путь + градиент).
