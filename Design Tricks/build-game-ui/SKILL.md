# build-game-ui

Сборка/репликация стилизованных UI-элементов в Figma (кнопки, инпуты, карточки, бейджи, панели,
табы, тогглы…). Главная ценность — **дерево решений**: элементов и вариантов много, собираются
по-разному, поэтому сначала классифицируем, потом выбираем приём(ы) из каталога и строим.
Точные рецепты и значения — в `references/`.

## Железное правило (нарушать нельзя)
**Бери факты из файла, не угадывай.** Прежде чем строить — сделай ДАМП реальных `fills`/
`opacity`/`blendMode`/`effects`/`strokes`/`cornerRadius`/`vectorPaths` оригинала через
`use_figma` и сними скриншот **в зуме на КОНТРАСТНОМ фоне**. Внешние тени, нижние блики, форму
текстуры легко не заметить в мелком тёмном тхамбнейле. Нельзя считать вариант «упрощённым» и
срезать объёмные приёмы — де-акцент это только палитра/насыщенность.

## Порядок действий
1. **Доступ/формат.** Есть Figma node-ссылка (читаю слои/значения) или только картинка? Финал — Figma.
2. **Разведка.** `get_metadata` → `get_design_context` → `get_variable_defs` → дамп слоёв
   (см. `references/figma-recipes.md` → «Дамп»). Скриншот в зуме на контрастном фоне.
3. **Классифицировать.** Определи ТИП ЭЛЕМЕНТА (см. «Типы элементов»), материал, структуру, слои
   глубины, форму текстуры, состояния, респонсив, контент-слоты. Выпиши, что **нового** (нет в каталоге).
4. **Архитектура до вёрстки.** Фон / декор-рама / эффект-FX → отдельные **компоненты**, переиспользуемые
   во всех состояниях. Элемент — **auto-layout**, размер **Hug** (или Fill по контексту), при
   растяжении — **3-слайс/9-slice** (центр FILL). Состояния — варианты компонент-сета.
5. **Итерация 1 (быстро, можно реюз).** Реюз внутренних компонентов/ассетов, собери, сверь скриншотом в зуме.
6. **Итерация 2 (сам).** Фон/декор векторами/слоями (`references/vector-method.md`) или сгенерированной
   картинкой (`figma.createImage`, см. recipes).
7. **Состояния + проверка** по чек-листу.

## Типы элементов (что добавляется к общим приёмам)
Приёмы T1–T15 ниже — общие. Специфика по элементам:
- **Кнопка** — лейбл (часто отдельный слой), состояния Default/Hover/Active/Disable/Loading.
- **Инпут/поле** — слоты: placeholder vs value (разный цвет/прозрачность), caret, иконка(и), хелпер/счётчик.
  Состояния: Default/Hover/**Focus** (рамка-акцент/glow)/Filled/**Error** (красная рамка+иконка)/Disable.
  Фон обычно «вдавленный» (inner shadow внутрь), а не «выдавленный» — инверсия T11.
- **Карточка** — медиа-слот (картинка/превью) + контент (заголовок/текст/цена) + бейджи/CTA.
  Состояния: Default/Hover (lift + усиление тени/рамки)/Selected (акцент-рамка)/Disabled.
  Часто рамка с вырезанными уголками (notch) — отдельный компонент-декор (T8/T9, негативное пространство).
- **Бейдж/чип** — компактная пилюля (T5), иконка+текст, иногда «новизна»/счётчик.
- **Панель/диалог** — крупная рама (T8) + фон (радиал/градиент/текстура) + заголовок; состояний обычно нет.
- **Таб/тоггл** — сегменты; активный = акцент-фон+глубина, неактивный = де-акцент; «бегунок» (T13-подобно).
- **Прогресс/слайдер** — трек (вдавленный) + заполнение (глянец/энергия T3/T7) + ручка.

## Дерево решений: какой приём выбрать
По признакам референса (комбинируются). Точные параметры — `references/techniques.md`.
- Прямые/скошенные края, тянется по ширине → 3-слайс (Left FIXED/Center FILL/Right FIXED) [T1].
- Фон — единый РАСТР, тянется → 9-slice растра `scaleMode:CROP`+`imageTransform`, общие доли швов [T2].
- Неон/стекло, объём у кромок → краевые alpha-градиенты + 30% wash + текстура [T3].
- Гладкий глянец → вертикальный градиент свет→тень + верхний блик + нижняя фаска + INNER_SHADOW [T4].
- Пилюля/чип, мягкий объём → жёсткая тень blur0 снизу + тонкий белый внутр. stroke [T5].
- Светящееся «ядро» → радиальный градиент + INNER_SHADOW [T6].
- Энергия/молния/дым → РАСТР + `LIGHTEN` + alpha-маска (не вектор) [T7]; анимация — FX/GIF на hover [T13].
- Металлическая рама-оправа → несколько концентрических rounded-rect с градиентами [T8]; рейлы — градиент-stroke.
- Орнамент/филигрань/notch по краям → отдельный вектор-компонент, зеркало [T9] (рисовать по vector-method).
- Блестящая поверхность → верхний глянец + диагональная засветка + INNER_SHADOW [T10].
- «Выдавить» из фона + низ → внешняя тёмная DROP_SHADOW (мягкая+плотная) + мягкий нижний блик [T11].
  (Для ИНПУТОВ — инверсия: «вдавить» внутрь, inner shadow тёмный сверху-внутрь.)
- Текст/лейбл должен «играть» → градиентная заливка текста + жёсткая тень 0/1 [T12].
- Состояния → один скелет, меняем свет/насыщенность/glow/FX; компонент-сет [T14] (семантика — см. «Типы»).
- Анимация только на hover → GIF-фон + контент слоем + прототип Default→Hover (ON_HOVER) [T13].
- Текстура из точек/стрелок → смотри ОБЩЕЕ (форма эмерджентная, бывает несколько крупных направленных
  форм), и она НЕ тянется (фикс. сетка `constraints:MIN`), база тянется отдельно [T15].

## Переносимые принципы
- Глубина = СТЕК слоёв (база → alpha-кромки → INNER_SHADOW → внешняя тень), не один fill.
- Единый источник света (верх светлее, низ темнее). Выпуклое = свет сверху; вдавленное (инпут) = тень сверху-внутрь.
- Негативное пространство — через `evenodd`/boolean/слой-снизу.
- Материал решает приём (металл = многостоповый градиент с резким свет→тень переломом).
- Тёплый металл × холодная энергия (золото/бронза + синий) = «дорого».
- Перекрас под тему = клон градиента + замена цветов стопов (тот же `gradientTransform`).
- Реюз > имитация: фон/декор/FX держим компонентами; перекрашиваем, не дублируем геометрию.

## Чек-лист сдачи (пройти ДО показа)
1. auto-layout? 2. размер Hug/Fill по контексту (высота паддингом, не Fixed)? 3. тянется по горизонтали (3-слайс/CROP)?
4. **фон/декор/FX вынесены в отдельный КОМПОНЕНТ и переиспользованы во ВСЕХ состояниях** (один imageHash/одна
   геометрия; состояния меняют только оверлеи/свет, не дублируют фон)? 5. глубина слоями (alpha-кромки/INNER_SHADOW/wash/тени)?
6. **внешняя тень-отрыв / нижний блик (или для инпута — «вдавленность») на месте** (частый промах)?
7. **форма текстуры повторена** (смотрел общее, не «горошек»)? 8. у нарезанного растра швы совпадают по пикселям?
9. шрифт/контент-слоты по характеру оригинала (placeholder≠value, иконки, бейджи)? 10. нет лишнего чёрного контура?
Любое «нет» — доделать.

## Грабли use_figma (критично)
Полный список и код-рецепты — `references/figma-recipes.md`. Самое важное:
- Ошибка в коде = **откат всей транзакции** → проверяй значения заранее.
- `use_figma` **не возвращает значения** → id новых нод узнавай через текст-слой + скриншот.
- `constraints` enum: `MIN|CENTER|MAX|STRETCH|SCALE` (НЕ TOP/BOTTOM).
- COMPONENT/COMPONENT_SET в `get_screenshot` часто 1×1 → снимай **инстанс**.
- GIF/PNG-фон: `upload_assets` POST из песочницы блокируется прокси → вшивай через `figma.createImage(bytes)`
  + ручной base64-декодер. ВАЖНО: `createImage`-заливка рендерится на ХОЛСТЕ, но НЕ в `get_screenshot`
  и НЕ в плеере ПРОТОТИПА. Если нужен прототип/Present — фон должен быть импортированной картинкой (ручной импорт).
