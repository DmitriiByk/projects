## ТОП-оптимизации (что стоило сделать раньше и сэкономить ходы)

1. **Сначала проверять local vs remote, потом строить.** Я построил полную миграцию (спеки, Tokens Studio JSON, 2 скрипта, ревью) в предположении, что «грязные» токены (`Pallette/*`, `Text/*`, `Buttons/*`) локальные. Дрифт-ран показал: они в ПОДКЛЮЧЁННОЙ библиотеке, а локально всё уже чисто. Потеря ~6–8 ходов. **Правило: первым делом `getLocalVariableCollectionsAsync` + `getLocalVariablesAsync`** — увидеть, что реально можно править, и только потом планировать. `get_variable_defs` врёт (отдаёт объединение local+remote).

2. **Для записи в Figma — сразу `use_figma` (MCP), не Scripter и не computer-use.** Я прошёл болезненный путь: Scripter (баги вывода `console.log` vs `print`, краш от рекурсивного шима, «нет лога» из-за floating-promise без `await`), потом computer-use (юзер не дал системные права). А `use_figma` выполняет тот же Plugin API напрямую от меня — создал 39 theme-токенов и привязал 33 текст-стиля двумя вызовами без единой ошибки. **Дефолт записи = use_figma.**

3. **Финализировать токен-шкалу ДО перепривязки.** Я перепривязал Discovery, потом доделал `tint/100/150/250` и полный `main` — пришлось гнать Discovery второй раз (+24). Сначала собрать всю семантику, потом один проход перепривязки.

4. **Блокеры выяснять в самом начале.** Вопросы «Enterprise? есть ли доступ к библиотеке? local или remote?» всплыли поздно. Они определяли всю стратегию (REST недоступен без Enterprise; библиотеку не мигрировать без доступа). Спрашивать такое — первым делом.

## Рабочие паттерны (переиспользовать)

- **Цвет-сохраняющая перепривязка.** Семантический токен алиасит ТОТ ЖЕ примитив → перепривязка с примитива на семантику не меняет цвет ни на бит. Гард: резолвить hex старого и нового, при несовпадении — пропуск (`vhex(cur)===vhex(tgt)`). 0 визуального риска, 0 skips = всё верно.
- **Деление с субагентом:** субагент в Figma ПИСАТЬ не может (нет MCP-коннекта). Схема: я выгружаю структуру через `use_figma` → агент строит judgment-карту (имена/маппинг) из данных → я применяю через MCP → верифицирую ридбэком. Отработало на: token-audit, спек-маппинг, ревью скриптов, карта переименований 58 компонентов. Это и есть «агент по именам», отдельный definition не нужен — хватает general-purpose с хорошим промптом.
- **Идемпотентность + верификация.** `use_figma` пишет сразу (нет dry-run) → код всегда идемпотентный (проверка существования перед созданием) и после записи — ридбэк (резолв алиасов/boundVariables, сверка hex/имён).
- **Большие чтения — скриптом, не дампом.** `get_metadata`/`index.html` переполняют контекст. Вместо дампа — `use_figma`, который возвращает компактный JSON ровно нужного (имена сетов, пропы, пути слоёв).

## Сборка/правка компонентов через use_figma (playbook, кейс 2026-06-12)
- **AUTO-LAYOUT СAЙЗИНГ — главная гоча.** Чтобы контейнер РОС при вставке детей (иконка в кнопку и т.п.), контент-фрейм должен быть `layoutSizingHorizontal="HUG"`, НЕ `FILL` и НЕ `FIXED`; и outer тоже HUG. FILL/FIXED → дети вылезают/обрезаются, фрейм не растёт. Стоило нескольких итераций — проверять это ПЕРВЫМ при «не растёт/обрезается».
- **Тумблер слоя в COMPONENT_SET:** (1) вставить слой во ВСЕ варианты (в контент-фрейм), (2) `set.addComponentProperty(name,"BOOLEAN",false)` → propId, (3) в каждом варианте `layer.componentPropertyReferences = {visible: propId}`. Default `false` → существующие инстансы АВТОМАТОМ получают слой выключенным (плюс sweep по инстансам для надёжности).
- **Цвет вставляемого элемента = как у текста:** читай переменную текста (`text.fills[0].boundVariables.color`) и привязывай к ней. Перекрашивай ВСЕ вектора иконки (`findAll(VECTOR)`, не `findOne` — глиф бывает из нескольких).
- **Размер под вариант:** парсь имя варианта (`size=md`), маппь на size-вариант суб-компонента, `component.createInstance()`.
- **Метод верификации правки компонента:** скриншот BASELINE до → правка → состояние «выключено» должно быть ИДЕНТИЧНО baseline (и размеры, и картинка) → тест «включено» по всем размерам → проверка целостности (каждый вариант имеет слой + верный componentPropertyReferences) → sweep инстансов. **Сначала тест на ОДНОМ варианте + скриншот, потом раскатка на все** — де-рискует.
- High-Custom (зелёная рамка): правки ВНУТРИ контент-фрейма (не трогая chrome-слои: градиент/обводку/gap) сохраняют вид. Структурные правки — только по явной просьбе владельца.

## Токен-доски (live showcase) через use_figma
- Свотч = frame, заливка привязана к переменной (`setBoundVariableForPaint`), подпись = имя + резолв алиаса в hex.
- **Градиенты = paint STYLES** (`getLocalPaintStylesAsync`), живая привязка через `node.fillStyleId = style.id` (или `setFillStyleIdAsync`).
- **Opacity-свотчи** не видны над белым → две заливки: тёмная solid снизу + привязанная полупрозрачная сверху.
- **Типографика** = текст-нода + `setTextStyleIdAsync(style.id)` (живой сэмпл реального начертания); предзагрузить все уникальные шрифты стилей.
- Раскладка — вложенные auto-layout (HUG), колонки по семействам/группам.

## Доп. гочи инструментов
- При дампе дерева guard-ить доступ к свойствам: у TEXT нет `primaryAxisSizingMode` (бросает) — оборачивать в try.
- `getNodeByIdAsync` по id стабилен даже после перемещения узла юзером — искать по id, не по координатам.
- `get_metadata` переполняет контекст на больших узлах → тащить компактные данные скриптом `use_figma`.

## Что НЕ было ошибкой (подтвердилось)
- AskUserQuestion на развилках — окупается, юзер ценит точечные вопросы.
- Независимое ревью отдельным агентом перед применением — поймало реальный блокер (коллизия Links/Alert) и важные баги по кнопке (перекраска не всех векторов). Кто делал — тот не проверяет.
- «Сначала один вариант + скриншот, потом все» + сверка с baseline — поймала проблему роста xs до раскатки боли на 40 вариантов.
